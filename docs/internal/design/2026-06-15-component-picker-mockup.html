The component insert picker — 2026-06-15 (rev. 2, opt-in preview)
The preview pane is the differentiator, and it is now opt-in and honest. Browse in one column. The configure step splits to a form plus a rendered preview only when a component declares a preview; otherwise it keeps today's single-column form. The preview renders the real directive through the site pipeline into a sandboxed iframe, the same as the editor, so it is async and debounced (it settles, it does not pulse), and it has real states: settling, render-failed, and incomplete. Panel 2 is live (debounced): edit Title or Tone to watch it settle.
1 · Browse the catalog (light) — one column, grouped, glyph + description + use
Components group under one-word headings (group) in declaration order, never alphabetical. Each row carries its glyph (icon), label, description, and intended-use, which is enough to recognize a block. No preview pane while browsing: over a handful of components a second pane is dead weight. Search appears above the list only once a site declares more than a named threshold (~8) of components.

2 · Fill it in, with preview (light) — the component declares a preview
Picking a component that declares a preview opens the two-pane configure step: the form on the left, the rendered result on the right. The right pane runs the real site render (debounced, latest-wins), so the chip reads settling, not a fake "live" pulse. Back returns to the catalog (one step). Required fields carry an asterisk and block Insert with inline text; repeatable rows show a derived label (itemLabel). Edit Title or Tone to watch the preview settle.

3 · Fill it in, no preview (light) — a component that declares no preview
Not every component has a meaningful visual, and the smallest catalogs do not need a pane. A component that declares no preview keeps today's single-column form, so the second pane never becomes dead weight. This is the default; the two-pane layout is the opt-in.

4 · The honest preview states — incomplete, render-failed
The preview never lies. Left: a required field is empty, so Insert is disabled and the preview shows the real skeleton with the empty region called out, never a fabricated "Untitled" finished block (the trap the design set out to avoid). Right: the render threw, so the pane says so and keeps the form intact, reusing the editor's existing render-failed surface.
