Media management — Direction B: the unified command surface
One insert entry point, routed by intent. The author types into a command surface (a combobox over a listbox) that filters across three lifecycle panes at once: Stored files (the R2-backed library), Embed (paste a URL, oEmbed), and Icon (curated design tokens). It opens three ways that all land on the same surface: slash at the caret, the Cmd/Ctrl+K palette, and a visible toolbar Insert button. The button is the rule that keeps a keyboard-first surface reachable: a hidden affordance strands the non-technical author, so the palette only ever shadows something visible. Locked to Warm Stone; built light, with dark intent noted. Screens 2 and 9 are live.

1 · Three doors, one surface — the fluent path and the visible fallback
The differentiator (5e, reuse) begins here: there is one place to insert anything. The editor toolbar carries a real, labeled Insert button (with its / cue), so a novice never has to know the shortcut. The same surface opens from a slash typed at the caret and from the Cmd/Ctrl+K command palette's "Insert media" command. The author picks by intent; the system routes to stored, embed, or icon.

2 · The command surface — type to filter across panes (live)
The heart of the direction. One combobox; the typed query filters every pane at once and the result list groups by lifecycle. Stored leads with the upload affordance (the visible backbone) and then matching assets; search runs over filename, alt, and caption. The scope chips above narrow to one pane (or stay on All). This panel is real: type in the box ("shoe", "trail", "winter", "youtube", "snow"), and use ↑ ↓ then Enter to move and choose. The selected row is announced through the live region. 5d (where-used) shows inline as a "used in N" pill; 5b (alt) shows as a "needs alt" flag right in the list.

3 · The insert step — alt text and name captured at insert (differentiators 5b, 5c)
Picking a stored file (or finishing an upload) lands on one compact step inside the same surface, reached by Enter or Tab. This is where the keyboard-first flow meets the two hardest-to-skip fields. (5b) Alt text is required, with an explicit Decorative escape hatch: you cannot insert until you choose one. (5c) A pasted screenshot is proposed a readable name from the post slug, with one rename field, so a bare image.png never lands. The preview matches what will render.

4 · Upload with no dead interval (differentiator 5a)
Drop, paste, or choose a file and an optimistic placeholder lands at the caret immediately, with determinate progress. No silent gap between the gesture and a visible result. On failure the same placeholder turns into an inline error with a Retry, never a silent drop. Three states, in the manuscript where the image will live.

The placeholder is a real decoration widget at the caret offset, swapped for the committed ![alt](media:…) on success. Each state maps to a log event (media.uploaded, media.upload_failed with a reason), so "my image didn't save" is greppable. The uploading row is an aria-live="polite" status; the failure is role="alert" so it interrupts. Progress percentages update the text, throttled, not on every frame.
5 · Dedup on upload — identical bytes resolve to the existing asset (differentiator 5e)
Upload a file whose bytes already exist and cairn does not make a duplicate. The content hash matches, so the surface resolves to the existing asset and says so. The author still gets the alt and caption step, now pre-filled from the asset they are reusing. This is the good silence the research calls for: no photo-2.jpg sprawl, no silent overwrite of different bytes.

6 · The other two panes — Embed (paste a URL) and Icon (pick a token)
The same surface, scoped. Embed: paste a video or post URL; oEmbed resolves the provider and shows a confirm card; nothing is stored, the content holds a directive. Icon: the curated design-token set, as a roving grid; never an upload. 11 · Accessibility — the focus order, the keymap, the narration, the visible fallback
A command-first surface is highest-risk for accessibility, so it is designed here, not bolted on. This panel is the spec the Svelte build implements.

The command surface is a combobox over a listbox

It follows the WAI-ARIA editable-combobox-with-listbox pattern. The text input owns role="combobox", aria-expanded, aria-controls pointing at the results, and aria-activedescendant pointing at the highlighted option. Focus stays in the input the whole time; the arrow keys move the active option without moving focus, so a screen reader announces each row as it becomes active. The results container is the listbox and each row is an option. The icon pane is a two-dimensional grid of gridcells, where arrows move in both axes.

The visible, non-keyboard fallback (the load-bearing rule)

Every action the keyboard reaches has a visible affordance a mouse or touch user reaches without a shortcut. The toolbar Insert button opens the surface (it is labeled, not a bare glyph). Inside, every option is a click target with a hover-revealed chevron, the scope chips are real buttons, and the insert step's fields are standard inputs with a visible Insert button. The slash and Cmd/Ctrl+K are accelerators layered on top, never the only door. A non-technical author who never learns a shortcut can do everything by pointing.

Keyboard map

/ at line start, or Cmd/Ctrl+K, or click InsertOpen the command surface. Focus lands in the input.
↑ ↓Move the active option in the listbox. Wraps at the ends. In the icon grid, ← → move too.
Ctrl+1 / 2 / 3Jump the scope to Stored, Embed, or Icon. The query is kept.
EnterChoose the active option. A stored file or upload goes to the name-and-alt step; an icon or resolved embed inserts.
TabFrom the input, move into the name-and-alt step for the active stored file (the same as Enter for that pane).
EscClose the surface and return focus to the caret (or to the Insert button if that opened it). The slash that opened it is removed.
Enter on the focal dot, then ←↑↓→Nudge the focal point 1% per press; Shift nudges 10%. The dot is a slider with aria-valuetext.

Screen-reader narration (what gets announced)

On open: "Insert media or a component, dialog. Insert a photo, paste a video link, or pick an icon, combobox, expanded. 12 stored files, 6 icons."
On typing "trail": a polite status says "38 results"; then arrowing announces the active option: "trail-map.pdf, PDF, 1.1 megabytes, used in 1 entry, option 1 of 38."
On an option that needs alt: the row's accessible name includes "needs alt text", so the gap is heard, not only seen as a yellow chip.
On upload: the optimistic row is a polite status ("first-melt-lake-loop-shore.webp, uploading, 68 percent"); a failure is an alert ("trail-map.pdf could not upload. The file is 24 megabytes. The limit is 20 megabytes. Retry button.").
On the delete gate: "This image is used in 3 entries, dialog. Type blue-running-shoes.jpg to confirm. Delete anyway button, dimmed." The button enables only when the typed name matches.

Contrast and motion

Muted text is the Warm Stone --color-muted token on the panel surface, never opacity stacked on muted (the design-system rule). The active option uses the primary 10% tint plus a 30% primary border, so selection is carried by border and background together, not color alone. Required-alt is flagged with the warning token plus the word "needs alt", so it does not rely on hue. The upload shimmer, the spinner, and width transitions are all behind prefers-reduced-motion. Determinate progress is text plus a bar, so it is legible without color.

12 · Dark — the command surface carries over
The same tokens flip to the dark theme root. The selection tint, the type badges, the usage pills, and the gradient thumbnails all hold on the dark card. Dark intent: the floating surface keeps the defined hairline border (a shadow barely shows in dark), and the active-option border stays at the primary-30% mix so selection is legible.
