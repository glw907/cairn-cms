Media management — the synthesized rev.2 (the design target)
The single direction the three-judge panel converged on, merging the best of insert-first, command-surface, and library-first. One insert entry point (a visible labeled Insert button, plus slash, paste, and drag) opens one at-caret popover that routes by intent: a photo or file opens the stored picker, a video or link opens the embed resolve, an icon opens the icon picker. The picker is built on the combobox/listbox accessibility spine (focus stays in the input; arrows move a live active-descendant). It is backed by a first-class Media management screen, a peer of Posts and Pages, with the same row design and affordances. Shared scenario: a posts-and-pages ski site, a 14-asset library with clean and messy names. Light theme is canonical; one dark panel proves the tokens carry. Panel 2 and the upload demo are live.

5 · Safe delete — the headline differentiator, gated by usage
cairn's content is markdown in git, so every reference is greppable. 5a deleting an in-use asset is a role="alertdialog" that lists every entry that would break, by placement kind, and requires typing the asset name to confirm. 13 · Accessibility — focus order, keyboard map, screen-reader narration

Designed-in accessibility for the core insert loop and the safe delete

The insert loop is keyboard- and screen-reader-reachable end to end: open the chooser, route to the picker, search or upload, capture alt, insert. The mouse path (the labeled Insert button, drop, paste) and the keyboard path (slash, the same button, the command palette) converge on one surface, so nothing is mouse-only. The picker is a conformant combobox over a listbox; the icon pane is its own 2-D grid; alt capture is a radiogroup; the grid on the management screen is a roving listbox of options, never list/listitem on interactive cards; and safe-delete is an alertdialog. Contrast holds the design-system floors; muted text is never stacked on opacity, and every state carries a glyph or a label, not hue alone.

The picker is a combobox over a listbox

The search input owns role="combobox", aria-expanded, aria-controls pointing at the results, and aria-activedescendant pointing at the highlighted option. Focus stays in the input the whole time; the arrow keys move the active option without moving focus, so a screen reader announces each row as it becomes active. The results container is the listbox and each row is an option. Two separate live regions keep the two announcements from colliding: a polite status for the result count, and a second polite region for the active row, so a count is never clobbered by a row narration.

Focus order — the core insert loop

  1. Open. The labeled Insert button carries aria-haspopup="dialog" and aria-expanded. Slash at the caret and the command palette open the same surface.
  2. Chooser. A role="menu" of three routes; arrows move, Enter activates, Esc closes and returns the caret to the document.
  3. Picker. Focus lands in the combobox input and stays there. Arrows move the active option (aria-activedescendant); the upload backbone, the pager, and Insert are Tab stops after the listbox. The grid is one widget, not a stop per tile.
  4. Capture card. On upload or paste, focus moves to the file-name field, then the alt radiogroup (arrow keys pick Describe or Decorative), then the alt field, then Cancel and Insert. Insert is disabled until alt is described or marked Decorative, and the disabled button names why.
  5. Return. On insert or close, focus returns to the editor at the caret, with the new reference selected so the next keystroke is in context.

The popover is a non-modal anchored surface, so focus is not trapped: Esc or a click outside restores the caret. The capture card, the safe-delete, and the mobile sheet are modal (role="dialog" / role="alertdialog"), so they trap focus and restore it on close, the design-system dialog recipe.

Contrast, state cues, and motion

Muted text is the Warm Stone --color-muted token on the panel surface, never opacity stacked on muted. The active option carries the primary 10% tint plus a 30% primary border, so selection is border and background together, not color alone. "Needs alt" is the warning token plus a triangle glyph plus the word, and "No references found" is a labeled pill, so neither relies on hue. Determinate progress is a bar plus a spoken percentage. The upload sheen, the spinner, and the caret blink are all behind prefers-reduced-motion. Usage copy reads "found in N entries" or "no references found", never "unused", and the index spans the published site plus every open edit branch and greps inline, hero, and gallery references; a raw-HTML reference is not greppable and is noted as such.
