Direction A · Library-first (asset manager)

Media, as a place you go

This direction makes the library the center of gravity. Media is a first-class screen in the admin, a peer of Posts and Pages: the hub where an author browses, searches, organizes, sees what an asset is used in, replaces it, and deletes it safely. Insertion does not invent a separate world. It opens a modal picker onto this same library (the WordPress and Webflow model), with an Upload tab and a Pick-existing tab.

The three lifecycles still meet at one door. The author picks by intent, and the system routes: a stored file goes to the library, a video or embed becomes a referenced URL, an icon is picked from the design system. This direction is built to be strongest where a large library lives or dies: at organizing at scale, and at the usage-and-safe-delete moment that every incumbent fails.

1 · The insert door, and the chooser that routes by intent
One affordance in the editor (the toolbar Insert media control, shadowed by paste, drag, and a /media slash accelerator) opens this chooser. The author names the intent, never the storage bucket. A photo or document routes into the library picker; a video or embed routes to the paste-a-URL flow; an icon routes to the design-system picker. The three lifecycles stay coherent at one door even though their backends differ.

4 · The five differentiators — the moments that beat the field
These are the moments the research says every incumbent fumbles. Each is rendered as a frozen frame of the live behavior. Differentiator (b), alt-at-insert, lives in panel 3a; (c), naming-at-insert, is the proposed-name field there and the paste case below. The signature of this direction is (d): usage tracking and safe delete, made cheap because cairn's content is greppable markdown.

5 · The three placements — one asset, three doors
A stored image gets used in three places, and they are three affordances on one asset. The reference is always the logical media: handle that resolves at delivery, never a raw URL.
5a · Inline body image — ![alt](media:…) in the manuscript, decorated to a widget
In the editor the image is plain markdown referencing the handle. The empty-bracket cue still works: a missing alt reads as ![]() and the bar flags it. The caret line shows the raw syntax; the surrounding lines show the rendered widget cairn decorates it to.

A11y · designed in — focus order, keyboard map, and SR narration for the primary flow
The primary flow is insertion through the modal picker, which the brief calls out as the surface that must be keyboard- and screen-reader-reachable with real focus management. This panel is the spec the implementation owes.

Focus order — open to insert

  1. Trigger (toolbar Insert media) opens the chooser. Focus moves to the chooser's first choice; the trigger is remembered.
  2. The three intent choices are a normal tab sequence. Enter on "a file you keep" opens the picker.
  3. In the picker, focus lands on the active tab (Upload). ← → move between Upload and Pick; the panel follows.
  4. Tab order then runs: dropzone (a real button) → Name → Alt mode (Describe / Decorative) → Alt textarea → Folder → footer Cancel → Insert.
  5. Focus is trapped inside the dialog. Esc closes it and returns focus to the toolbar trigger.
  6. On Insert, the dialog closes, focus returns to the editor at the caret, and the new image's edit handle is reachable with one Tab.

Keyboard map

/mediaSlash accelerator: open the chooser at the caret (shadows the visible button).
EscClose any dialog; return focus to the opener. In the safe-delete dialog it cancels.
← →Move between picker tabs; move the focal-point dot by 1% (with Shift, 10%).
↑ ↓Move the focal-point dot vertically; move selection in the grid by a row.
SpaceToggle a card's checkbox in the grid (multi-select for bulk ops).
EnterOpen the focused asset's detail, or activate the focused button.
⌘/Ctrl+ASelect all on the current page (announced as "N selected"), never silently across pages.

Screen-reader narration — the primary flow, start to finish

[open] "Add media, dialog. A file you keep, button. Upload it or pick one already in your library. One of three."

[picker] "Add a file, dialog. Upload, tab, selected, one of two. Drop a file to upload, button."

[upload starts] aria-live polite: "Uploading shoes dash closeup dot jpg, 68 percent." Then "Stored. Name, edit text, best dash trail dash shoes dash closeup."

[alt gate] "Alt text, required. Describe it, toggle button, pressed. Edit text. Required, or mark it decorative for an empty alt." Insert stays disabled and is announced "Insert, dimmed, alt text needed before insert."

[decorative path] Activating Decorative announces "Decorative, pressed. Alt text not needed for a decorative image," and Insert enables: "Insert, button."

[insert] aria-live polite: "Inserted blue dash running dash shoes. Back in the editor." Focus is at the caret, the image handle one Tab away.

[safe delete, the signature] The dialog is an alertdialog labelled by its title and described by the warning. "Delete blue dash running dash shoes dot jpg, alert dialog. This file is in use. Deleting it will break the image in these entries when you publish. Used in 3 entries, list, 3 items. Best trail shoes for spring, inline image. Gear we love this season, hero image. Shop, gallery tile 2. Type delete to confirm, edit text." Until the word matches, "Delete anyway, dimmed."

Contrast and non-color cues

  • Every state pairs color with a glyph or text: the usage pill carries a count, the orphan pill carries "Unused," the type pills spell DOC / EMBED / ICON / PDF.
  • Muted text is a single token (--color-muted / --color-subtle), never opacity stacked on a tint, so it keeps its measured contrast.
  • Selection is an outline plus a filled checkbox, not color alone; the focal dot has a white ring on a dark scrim for contrast over any image.
  • Required fields carry an asterisk with aria-required and gate the action with an inline message, not a color shift alone.
  • The filename stamp on each swatch is a mockup stand-in for the real bytes; in production the thumbnail is the image, with the name in the card label below it.
  • Progress is a determinate progressbar with aria-valuenow, and its percentage is also spoken through an aria-live region, never color alone.
