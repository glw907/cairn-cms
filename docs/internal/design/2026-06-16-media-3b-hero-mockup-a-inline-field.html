Media 3b, the hero image, direction A: the consistent inline field

A concept can carry a hero image in its frontmatter as a nested object, image: { src, alt, caption }. The field lives in the editor's details slide-over, alongside the other frontmatter fields. This direction keeps the hero at the visual weight of its neighbors, so the panel stays calm and uniform. It reuses the 2b capture and picker components, the same alt model (describe or decorative, alt as debt not a save block), and the resolved-thumbnail treatment. One image serves both the hero render and the social card, so there is one thing to set. The site template owns the hero layout on the page, so this field has no per-image placement role.

Light: the empty state and the set state, in the details panel

Left, no hero yet: a slim labeled dropzone sits between Description and Date, reading as one more field. Right, a hero set: a small thumbnail with the name, the alt state, an optional caption, and quiet Replace and Remove links. In both the field never overpowers the panel.

Light: alt is debt, and the decorative choice

Alt text never blocks a save. A hero set with empty alt shows a needs-alt marker in the warning ink, and the publish notice counts it. Edit opens the same describe-or-decorative choice the 2b capture card uses. A decorative hero is an explicit choice, so the field says so plainly.

Light: the chooser (the 2b insert surface, upload first then the library)

Add a hero image or Replace opens the same chooser the body-image insert uses: upload sits first, the combobox library picker below it. A drop straight onto the field skips this and goes to the capture card. Picking a library image with no alt carries its needs-alt marker through to the field.
