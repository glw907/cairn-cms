Media 3b, the hero image, Direction C (the capture-card echo)

A concept gets a hero image in frontmatter, edited in the editor's details slide-over alongside the other frontmatter fields. The stored value is a nested object, image: { src: media:slug.hash, alt, caption }. One image serves both the page hero and the social-card og:image. This direction reuses the 2b insert flow to the letter: the empty state is the 2b chooser (upload first, then the "or reuse an image" combobox), and the set state is the 2b MediaCaptureCard (thumbnail, the alt radiogroup, joined here by the caption field). An editor who has inserted a body image already knows this surface. The hero adds one move a body insert does not need: once captured, the tall card settles into a compact resting row so it sits quietly among the other fields.

Light, just captured: the 2b MediaCaptureCard, with the caption joined in

This is the 2b capture card almost verbatim. The thumbnail, the editable Name with its Suggested tag, and the alt radiogroup are the same controls a body insert shows. The hero adds the optional Caption field and a quiet line noting the image doubles as the share card. Save is never blocked on alt. When the editor is done, Done collapses the card to the resting row below.

Light, settled: the captured card collapses to a compact resting row

Once Done is pressed the card settles to this row, so the hero no longer dominates the panel. The row carries the thumbnail, the name, and the alt status. Edit reopens the same capture card. A caption, if set, previews beneath the row. This is the one thing the hero needs that a body insert does not: a body image lives in the manuscript, but a hero lives in this panel for the whole session, so it has to learn to sit quietly.

Light, settled with alt still missing: the resting row carries the needs-alt marker

Alt is debt, not a save block. A hero saved without alt still settles to the resting row, but the status reads "Needs alt" in the warning ink, and the same panel-level notice the editor uses surfaces below it. The decision stays with the editor; nothing here stops a save or a publish.

Dark, just captured: the capture card, decorative chosen (no alt needed)

The decorative path: when an image carries no meaning for a reader, the editor marks it decorative and no alt is owed. The needs-alt status never fires for a decorative hero. The caption stays available, since a decorative image can still carry a human-facing caption.
