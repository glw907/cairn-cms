Media 3c, the Media Library — Direction A: the office list, for media

The Library is a peer screen of Posts and Pages, reached from the same nav. Direction A mirrors the existing office list exactly so the screen feels native on sight: the eyebrow-and-count header, the search, the triage segmented control, the self-describing rows, the managed pagination, the details slide-over, and the safe-delete dialog all carry over from ConceptList. The rows borrow the 2b MediaPicker feel: a thumbnail, the name, an alt-status chip, and a usage pill. cairn's headline payoffs ride the row and the detail: where-used is exact and visible, and it gates the delete.

1 — Light: the library list

Each row describes itself: thumbnail, name, a metadata line (dimensions, size, type, date), the alt-status chip (glyph plus label, never hue alone), and the usage pill. valley-ridge reads Needs alt in the warning ink; old-trailhead-map shows no references found, the cue that it is safe to delete. The triage segments filter to All, Needs alt, and Unused with live counts; the type facet (Images / Documents) appears because the library holds both. A click on a row opens the details slide-over.

2 — Light: the details slide-over (where-used and actions)

A click on a row opens the details slide-over (the editor's details-panel recipe). It holds a larger preview, the full metadata, and the headline Where used list: each entry that references the asset is a link, tagged with the branch it was found on (the cross-branch grep reaches open cairn/* edit branches, not just main). Beneath, the alt and the name are editable in place, with Replace and a Safe-delete control. The list behind the panel dims under a light scrim, and the row that opened it stays selected. Editing here commits on Save, the same as the editor.

3 — Light: the safe-delete alertdialog (in-use and orphaned)

Delete opens a type-to-confirm alertdialog, never a plain confirm, because deleting media can break pages. Left, the asset is in use: the dialog names the entries that would break and requires typing the asset name to enable Delete. Right, the asset is orphaned: no references found, so the dialog is a light confirm that notes git history makes it recoverable. Neither dialog light-dismisses on the backdrop.
