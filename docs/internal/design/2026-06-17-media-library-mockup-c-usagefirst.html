Media Library exploration C: usage-first / cleanup-forward

The bet of this direction: make cairn's structural superpower the spine of the whole screen. Every rival's number-one missing or paywalled feature is "where is this used" plus a safe delete. cairn knows usage exactly, for free, by grepping media:<slug>.<hash> across main and every open cairn/* edit branch. So the library leads with usage and alt-debt rather than treating them as detail-panel footnotes. A triage header counts what is worth acting on, a Tidy up lens surfaces the orphans and the missing alt first, usage is the headline column on every row, and the detail view is built around the where-used list. Safe-delete is the star interaction.

1. Light: the default view, usage-forward, with the triage header

Media sits in the nav as a peer of Posts and Pages, so the office shell stays (sidebar plus topbar). The triage header is the headline read: total assets, then the two counts worth acting on (unused and needs alt) in the warning ink, then storage used. The All lens is active here, so every asset shows. Usage is the leading data column: a green "Used in N" or a muted "Not used", each with a one-line locator. The meadow-fence row is an orphan and valley-ridge needs alt, so both stand out without a separate screen. Load-more sits at the foot, never infinite scroll.

2. Light: the Tidy up lens (orphans and alt-debt first)

The Tidy up lens narrows to the assets worth acting on, orphans and missing alt, ordered with the orphans on top. The explainer strip names what the lens does and what makes a delete safe here (git history). An orphan row leads with "Not used" and carries the delete action inline. A used-but-needs-alt row keeps its usage count (so the author sees deleting it would break a page) and offers Add alt. When the lens empties, it reads as an all-clear rather than a bare blank.

3. Light: the detail view, built around where-used

Opening an asset slides a panel over the list (the row stays selected, with the left accent edge and an announced selection). The where-used list is the centerpiece, directly under the identity header, grouped by published versus open edit branch since cairn unions main with the open cairn/* branches. The first group is live on the site; the second is a draft on a per-entry edit branch that has not been published yet, named by its branch. Metadata, the alt and rename editors, Replace, and a Delete that opens the safe-delete dialog sit below.

4. Light: the safe-delete alertdialog (the star interaction)

Two faces of one dialog. Left, an in-use asset: it lists the exact entries that would break, grouped the same way (published first, then the named edit branch), and requires a typed confirm so the delete is deliberate. Right, an orphan: a calm confirm that states nothing references it and that git history keeps the bytes recoverable, so there is no fear and no typing. Both are role="alertdialog" with an explicit confirm, no light dismiss.
