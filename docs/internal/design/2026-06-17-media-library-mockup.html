Media 3c, the Media Library (consolidated, final)

The Media Library is a first-class admin screen, a peer of Posts and Pages, reached at /admin/media. Its only job is to manage the site's committed media: browse every asset, see where each is used, fix its alt and name, replace it, and delete it safely. The public site renders no gallery. This is the consolidated design, after the three exploration mockups and the adversarial critique: a contact-sheet grid at full content width is the default, the detail opens in a non-modal slide-over, a density toggle flips the grid to an enriched table for documents and for sort-and-scan, where-used rides every asset and gates delete and replace, and safe-delete is a two-faced alertdialog. Multi-select, bulk actions, and folders are out of scope by decision.

1. Light: the grid default view

The header is the office recipe: the Media eyebrow, a Media library h1 in the display face, a live count line, and Upload top-right. One toolbar row carries search (left, searching name, alt, and caption), the triage segmented control (All, Needs alt, Unused with live counts, a radiogroup with aria-checked), and a grid/list density toggle (right). The grid is a roving-tabindex listbox: each tile is an option with object-fit: contain on a quiet mat, the name always in a footer, and the alt-status as a glyph carrying an accessible name. A needs-alt tile shows the warning glyph; an unused tile carries the corner Unused marker. A managed Load more sits below with an announced count. Never infinite scroll.

2. Light: the list density (the enriched table, with a document and the sortable header)

The density toggle flips the grid to a real <table>. Each row carries the thumbnail, the name with a metadata line (dimensions, size, type, date), the alt-status chip, the usage pill, a sortable Added column (a real <th><button> with aria-sort), and an always-visible per-row delete. A trailing Upload media foot-row sits at the bottom. This density is the home for documents: old-trailhead-map is a PDF, so its alt status reads Not applicable and the row shows a document glyph. The triage and type facet show because the library now holds more than one type.

3. Light: the detail slide-over (where-used grouped, alt editor, actions)

Opening a tile slides a non-modal panel in from the right (the editor's details-slide-over recipe: no scrim, the grid stays live behind it, Escape closes it, focus returns to the originating tile). Top to bottom: a large preview, the name and the media: reference, the alt editor (the 2b describe-or-decorative radiogroup plus the alt field, alt-as-debt, never gated), Where used grouped by Published on the site versus In an unpublished edit (each entry a link; edit-branch entries name their branch), the metadata grid, and Replace and Delete.

4. The safe-delete alertdialog (in-use and orphan)

Two faces of one role="alertdialog", neither light-dismissing. Left, the in-use face: it names the entries that would break (grouped published-then-branch) and requires typing the slug to confirm, in a plain two-sentence voice. Right, the orphan face: a calm confirm, no typing, with the git-history reassurance. The where-used count warns in both. Copy says "found in N entries" and "no references found", never the bare word "unused" in prose.

4b. The replace-in-place dialog (usage-aware, a quiet slide-over action)

Replace opens a quiet dialog from the slide-over. It carries the same usage-aware language as delete, and one plain note: the placements keep their reference, so a swap reaches every entry, but the new image may differ in dimensions. The chooser is the 2b insert surface (upload first, then the library picker).

5. The empty state

A library with no media yet drops the grid and centers the office empty-state recipe on the content area: the cairn mark, the concept-named line, the Upload CTA, and a dropzone line. The triage segments and the search stay hidden until there is content to triage.

7. Narrow viewport: the grid stacks, the detail rises as a bottom sheet

Below the narrow breakpoint the slide-over cannot sit beside the grid, so the grid takes the full width at two columns and the detail rises as a bottom sheet (the dialog-sizing rule). The sheet carries the same preview, alt editor, where-used, and actions. Tapping a tile raises it; the grip or the close control lowers it, and focus returns to the tile.

The synthesis, in one line. The grid is the room and the slide-over is the desk: media is visual, so the picture leads at full width, and the management facts (alt, where-used, replace, delete) ride a non-modal panel one tile away, with a table density for documents and for sort-and-scan. cairn's structural payoffs carry the screen: where-used is exact and cross-branch, it gates delete and replace, and safe-delete trusts the author with the facts and a typed confirm only when a delete would break a published or drafted entry. Multi-select, bulk actions, and folders are deliberately out. The type-facet seam waits, hidden, until a second kind of file arrives.
