Media Pass B, Replace and Alt propagation (comparison direction)

Two operations the author starts from a media asset's detail slide-over. Replace swaps in a newly uploaded file; because cairn is content-addressed, the new file gets a new hash and every published reference is repointed to it in one commit to main. Alt propagation pushes the asset's edited default alt to placements that lack alt, with an opt-in to also overwrite placements that carry custom alt. Both rewrite N entries at once, so each opens a two-pane confirm modal: the left anchors the single change, the right lists every affected placement with its own before and after. A report-only branch-delta names open cairn/* edits that still point at the old asset and are never rewritten. Replace fails closed if usage cannot be fully verified.

Light: the slide-over at rest, where both flows start

The existing non-modal detail slide-over gains two openers. Replace is a quiet text control in the action row beside Delete. Push to placements sits under the default-alt editor, because it acts on that field's value. Both open the comparison modal; the slide-over stays where it is behind the modal scrim.

Light: Replace fails closed when usage cannot be fully verified

If a branch is unreadable or the index is stale, the impact is not fully known, so Replace will not proceed. The banner names the gap, the action stays unavailable, and the typed-name gate is hidden (there is nothing safe to confirm). Retry re-runs the scan.

Light: Push alt to placements, the three buckets

Left, the anchor: the old default alt over the new one, with the asset's thumbnail and name so the author keeps the image in view. Right, the placements split into three buckets. Will fill is every empty-alt placement, written by default. Already described keeps its custom alt unless the author opts a row in with its checkbox (Select all sits in the bucket header, and the bucket carries a count). Skipped is the decorative hero, left alone. The caveat states the one honest limit: a body image cannot record a decorative choice, so an empty-alt body image reads as a fill gap.
