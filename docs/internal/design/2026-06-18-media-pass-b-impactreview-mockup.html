Media Pass B — Replace and Alt-propagation — Dedicated impact review

Two new operations on the existing Media Library detail slide-over. Replace uploads a new file for an existing asset; cairn is content-addressed, so the new file has a new hash and every reference in published content is repointed to it in one commit to main. Alt propagation pushes the asset's default alt to placements that lack it, with an opt-in to also overwrite placements that carry custom alt. Both rewrite N published entries at once, so the screen's whole job is to make the impact legible and confirmable before applying. This direction puts that weight in a dedicated review modal, in the spirit of the existing safe-delete alertdialog. The resting slide-over stays simple and gains only the two entry-point controls. Read the thesis and the tradeoff in the source comment at the top of this file.

1 · Light: the slide-over at rest, with the Replace and Propagate entry points

The resting panel is the shipped detail slide-over. Two controls join the action group at its foot: Replace the file and Push alt to placements. They are quiet by design; neither does anything until its review modal is read and confirmed. Replace carries the danger border because it repoints a hash; Propagate is a primary-tinted everyday action. The grid behind stays live and in the a11y tree (the non-modal region recipe).

2 · Light: Replace — the impact-review modal

After the author uploads the new file, Replace opens this modal. It leads with the plain headline and the from/to identity strip: old bytes to new, with the unchanging media: reference called out (the reference is what keeps every link working). Then the affected-entries list (each row names the entry; the detail is just "repoint", since the alt at each placement is untouched by a file swap), the report-only branch-delta, and the typed-slug confirm that matches the safe-delete gate. The badge and the confirm are danger-red, because a replace repoints a hash and can change what a draft renders.

3 · Light: Push alt to placements — the three buckets

Propagation groups every placement into three buckets. Will fill (empty alt) is the default work and is always applied. Has custom alt is opt-in: a checkbox, a count, and a sample of exactly what gets overwritten, so the author sees the cost before choosing it. Decorative hero, skipped is reported and never touched, per the persisted decorative flag. The headline is tuned to the work, not to danger: "Fill alt on 5 placements". The footer count tracks the opt-in live (5 now, 8 if the custom box is checked).

5 · Light: Replace — fail-closed (usage could not be fully verified)

A replace rewrites every reference, so it must know every reference. If the usage scan cannot complete (a branch read failed, the cross-branch index is unverifiable), the dialog refuses rather than guess. It shows what it could and could not read, and offers a retry. Delete already fails closed the same way; Replace carries the higher stakes (it rewrites bytes), so the same fail-closed rule binds it. The confirm is gone: there is nothing safe to confirm.
