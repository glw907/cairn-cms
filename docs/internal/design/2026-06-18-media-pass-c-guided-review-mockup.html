Media Pass C — the destructive cluster — "guided review modal" direction

The whole destructive cluster on the existing Media Library at /admin/media. Three jobs. Bulk delete is ambient: a multi-select grid, a sticky "N selected" action bar, and a bulk delete that applies the single safe-delete gate per item (in-use items are skipped and reported, never force-deleted). Orphan collection is on-demand: a Find unused media action opens a guided review, a large native <dialog> that reuses the Pass B review-modal grammar (the ~38rem modal-box scroll container, the report-only dashed sections, the typed-slug-style confirm reserved for the irreversible). It presents two directions with different friction tiers: orphaned bytes (R2 objects with no manifest row, the lower-friction tier, behind the one typed confirm because the byte purge is irreversible) and unreferenced assets (manifest rows referenced by nothing, the strict fail-closed tier, a plain confirm because the removal is a revertible git commit). The shipped Unused facet is renamed No references found with the blind-spot caveat. The thesis: reuse the shipped Pass B confirm-modal grammar for the sweep so the destructive review feels like the rest of the media UI, on-demand rather than a standing screen. Read the decisions in the source comment at the top of this file.

1 · Light: the Library in select mode — the multi-select grid + the sticky action bar

Select mode is on (the toolbar's Select toggle). Each tile gains a real checkbox affordance top-left; selection is decoupled from focus (a tile can be focused without being selected, and the selection ring is separate from the focus ring). The triage facet's third segment is renamed from Unused to No references found. The sticky bar names the count and its explicit scope: "3 selected, the ones you picked in this view", with a quiet convenience link to select all matching the current filter. The bulk Delete is the danger-soft control; the heavier register lives in its confirm dialog. Find unused media sits in the office header beside Upload, the on-demand entry into the guided review.

grid is role="listbox" aria-multiselectable="true" (WAI-ARIA APG) · roving tabindex: one tabstop · Space toggles selection (decoupled from focus) · Shift+Arrow extends a range · Ctrl/Cmd+A selects all in view · Esc clears the selection · a separate sr-only role="status" announces "3 selected"

2 · Light: bulk delete — the skip-and-report confirm

Pressing Delete 3 opens the confirm. This is the single safe-delete gate applied per item, never weakened. The dialog splits the selection into will delete (no references found) and kept, still referenced (an in-use item, skipped with its where-used reason). The in-use item is never force-deleted here: forcing an in-use delete stays the deliberate single-item typed-slug path. The safety-floor line states the honest contract: the displayed split is from the quick index, and each item is re-checked with its own strict read at the moment of delete. The removal is one revertible git commit, so this is a plain confirm naming the count, no typed gate.

role="alertdialog" · showModal() (native focus trap + Escape) · NO method="dialog" backdrop (no light dismiss on a destructive surface) · initial focus is Cancel · the in-use SKIP is the safe-delete gate per item, never a bulk force-delete

4 · Light: Find unused media — the guided review (both directions, dry-run, caveats)

Find unused media opens the guided review, the on-demand sweep in the Pass B modal grammar. It presents the two directions as itemized sections. Direction A, orphaned bytes: files in storage with no manifest row, the lower-friction tier, each candidate stating its R2 key as the "why". Its one caveat is the raw-HTML case, and its purge is irreversible (the typed confirm is in the next frame). Direction B, unreferenced assets: manifest images referenced by nothing across every branch, the strict, per-item tier; each row carries a per-item re-checked cue (cairn re-read that exact image clean), and the section restates the blind-spot caveat inline. Direction B's removal is a revertible git commit, so it earns a plain confirm naming the count. The dry-run is mandatory: nothing runs until this review is read and confirmed.

role="dialog" (the review is the everyday register; the irreversible byte purge carries its own typed gate, the next frame) · the modal-box is the ~38rem scroll container · each Direction-B row's "re-checked" cue stands for a real per-item strict buildUsageIndex read · the two confirm tiers are deliberately different

5 · Light: the guided review — the irreversible byte purge (the one typed confirm)

Direction A's bytes are the one irreversible action, so it gets the one typed confirm, and it is kept visually and verbally distinct from the reversible git-tracked removal (avoiding the Lightroom Remove-vs-Delete-from-Disk trap). The byte purge section's own confirm block is in the danger family, states plainly that this clears the bytes permanently and there is no git copy of an orphaned file, and gates the purge behind a typed phrase. Direction B (the reversible images) keeps its plain confirm above. The two are never one button.

the typed gate is reserved for THIS irreversible action only · the purge button is disabled until the phrase matches · the reversible Direction-B confirm is a separate plain button, never folded into the typed gate

6 · Light: the guided review — Direction B on hold (a branch would not read), Direction A clean

The strict tier fails closed honestly. If a branch read could not complete, Direction B refuses to remove anything, names the unreadable branch, and offers a quiet Check again, the same fail-closed grammar as Pass B's blocked replace. There is no Remove button while it is on hold. Direction A here found no orphaned files, so it reports a calm clean state rather than an empty list. A direction is only ever as confident as its own read.

the on-hold banner is role="status" on open (no action attempted yet); role="alert" is reserved for a failure surfaced AFTER a Remove was pressed · no Remove button while Direction B is on hold
