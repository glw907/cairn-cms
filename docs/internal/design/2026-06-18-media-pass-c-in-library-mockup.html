Media Pass C — the destructive cluster, in the Library (the lean integration)

No new screen. The destructive cluster is layered onto the shipped Media Library at /admin/media as filtered bulk-select with honest confirms. The grid and the table become aria-multiselectable; a sticky N selected bar carries the explicit scope and the destructive action; the triage radiogroup gains two reconcile facets. Bulk delete is the single safe-delete gate applied per item: each hash re-runs its own strict cross-branch usage read and fails closed, an in-use item is skipped and reported, and the op ends with an itemized summary. Orphan collection is two facets, not a second screen: Orphaned files (R2 bytes with no manifest row, the irreversible byte purge) and No references found (manifest rows referenced by nothing, the reversible git removal). The "Unused" facet is renamed No references found with the raw-HTML blind-spot caveat. The safety floor is shown honestly: the displayed usage index is non-strict, so a select-all is a convenience and every item is re-gated at execution. cairn is not WYSIWYG: every before/after is a structured fact, never a rendered-page canvas. Read the decisions in the source comment at the top of this file.

1 · Light: the Library, multi-selectable, nothing selected (the renamed facet in the triage)

The shipped grid, now multi-selectable. Each tile carries a real native checkbox at top-left, decoupled from the roving focus: arrows move focus, Space toggles selection, focus and selection are independent (the WAI-ARIA APG multi-select model). The triage radiogroup is broadened from three segments to five: the shipped All / Needs alt, then the renamed No references found (was "Unused"), and the two new reconcile facets No references found and Orphaned files. With nothing selected there is no action bar; the surface reads exactly as it does today. The top-right usage chip is renamed too: "Unused" becomes No refs.

5 · Light: the Orphaned files facet — R2 bytes with no manifest row (Direction A)

This facet is the destructive half of the read-only reconcile, Direction A. These are R2 objects with no manifest row, so they are not in the asset list (no row, no tile): the facet renders a distinct byte-row section keyed on the R2 key, the bytes, and a last-seen, never a display name. The dashed preamble names why they are candidates and what cairn cannot see (a raw-HTML hardcoding the delivery URL). Because there is no manifest row to revert, purging the bytes is irreversible: the preamble carries the danger register and the sticky bar's action is the solid danger fill, verbally distinct (Purge 2 files). Direction A is the lower-friction tier in detection, but its action is the strict one, so it is the single place a typed-confirm gates the purge.

these rows are NOT role="option" assets; they are a separate aria-multiselectable list role="listbox" aria-label="Orphaned files" · same real-checkbox + roving model · the action is verbally distinct ("Purge ... files") and the only typed-confirm path in the cluster

6 · Light: purge orphaned files — the irreversible confirm with the typed gate

The byte-purge confirm is the one place in the cluster that reserves a typed confirm, because it is the one irreversible action. It lists the keys and bytes to remove (the dry-run), states plainly that this cannot be undone and that there is no git history for raw bytes, restates the blind-spot caveat at the point of action, and gates the purge behind typing the file count. The title and the action stay verbally distinct from a normal delete: Purge, never "Delete". The badge and button carry the solid danger fill.

8 · Light: bulk delete — on hold (a usage read could not be verified)

The safety floor in action. When cairn re-checks each item at delete time, a branch read can fail. Rather than guess, the op holds before touching anything. It names what read cleanly and the specific unreadable branch, exactly like the single safe-delete and the Pass B replace. There is no delete button, not even a disabled one; a quiet Check usage again re-runs the read. This is the per-item gate made visible: a bulk select never overrides it. Reached either before the op starts (the pre-flight read failed) or mid-run (a later item's read failed, after the summary already shows the items that finished).
