State 1 · The desk at rest — spellcheck on by default
The everyday writing state, the gold-standard desk shell unchanged. Spellcheck runs locally and is on by default, so the one real misspelling (recieve) carries a wavy underline. Nothing else is touched: the code span, the media: token, the frontmatter keys, and the :::figure directive are masked from the checker through the Lezer tree, so they are never flagged. Tidy sits in the toolbar as a quiet labeled control, waiting to be invoked.
Spellcheck = a @codemirror/lint source backed by a WASM dictionary on a Web Worker. The lint source reads the Lezer markdown tree and skips CodeBlock, InlineCode, URL, FrontMatter, and the cairn media:/cairn: token nodes, so machinery is never checked.
a11y: the wavy underline is a distinctive non-color decoration; the diagnostic is also reachable by keyboard (the lint cursor) and read by the screen reader as a diagnostic, never signalled by hue alone.

State 2 · Spellcheck — the correction popover
Click (or the keyboard fix command) on the underlined word opens a small anchored popover at the word. It carries the suggestions top-to-bottom (the first is preselected, number keys apply), then a footer with Add to dictionary and Ignore. Picking a suggestion replaces only that word; the author never leaves the line. The personal dictionary is per-site and persists, so names and product terms stop being flagged once added.
a11y: the popover is a real anchored dialog opened at the caret (the MediaInsertPopover precedent). Focus moves into the suggestion list, Up/Down walk it, Enter applies, Esc closes and restores focus to the word in the editor. aria-activedescendant tracks the highlighted suggestion.
"Add to dictionary" appends the word to the per-site custom dictionary (a git-committed file, consistent with content-in-git); "Ignore" suppresses it for this session only. Both re-run the lint source so the underline clears.

State 3 · Tidy review — in-flow inline track-changes
Tidy has run and now owns the buffer for review. Its changes appear right where they belong, inline in the markdown. A deletion is the original text struck through (kept until you accept), an insertion is the new text added beside it. This is a light copy-edit: a grammar fix (recieve to receive and the agreement that follows), a consistency fix (the author wrote cm and centimetres both ways, harmonized to their dominant cm), and a typo (a doubled the the). Word choice, rhythm, and the author's -8 to -14 C are left alone. The compact bar floats low, states the count, and drives a per-change accept/reject and a step-through. Notice the media: chip and the :::figure rails sit clean inside the review: a tidy that touched a token or a fence would be rejected before it was ever shown.
The model returns a rewritten STRING (poplar's model: string-in, string-out, no JSON patches). The client computes an LCS diff against the captured pre-tidy body and renders the changed runs as decorations. Insertions are CM6 decorations; deletions are widgets holding the removed original (it is no longer in the doc). Accept-all applies in ONE batched transaction.
Structure gate (make-or-break): before the review is offered, the result is validated as a proofread, not a restructure. The frontmatter, every media:/cairn: token, the :::figure fence, and the inside of the code span must be byte-identical; any drift fails validation and the review is not shown. That is why those spans carry NO change marks here.
a11y: the floating bar is role="toolbar". Keyboard: n/p step to next/previous change, y/n (or the bar buttons) accept/reject the focused change, A accept-all, Esc end review. A polite aria-live region announces each step ("Change 2 of 5: replacement, 'recieve' becomes 'receive'"). The active change carries a 2px ring AND the kind word, never hue alone.

State 5 · Settings — spellcheck on by default, tidy opt-in
Where the two features are turned on and configured, in the admin's Settings. The split is the locked principle: spellcheck runs locally so it leaks nothing and costs nothing, which is why it ships on by default; tidy sends content to the Anthropic API and costs tokens, so it is a deliberate opt-in the site owner turns on and supplies a key for. Spellcheck carries the personal dictionary (a git-committed per-site file). Tidy carries the API-key field, the model choice (Sonnet by default for the judgment a light copy-edit needs, Haiku as the cheaper option), and the plain privacy note.
The tidy toggle gates the Worker action and the toolbar entry: with tidy off, the Tidy button does not render. The API key is a Cloudflare Worker secret (never shipped to the browser); the field shows it masked with a live "key works" probe, mirroring the doctor --probe pattern.
