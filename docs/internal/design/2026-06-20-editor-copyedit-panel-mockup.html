State 1 · The desk at rest (spellcheck on, tidy dormant)
The everyday writing state. Spellcheck runs locally on a Web Worker and marks one misspelling (recieve) with a wavy underline in the warning ink. It is markdown-aware: the code span, the link URL, the frontmatter keys, and the media: token are never flagged. Tidy sits in the toolbar as a labelled control, dormant until invoked. The footer shows the quiet spellcheck count, not a red alarm.
Spellcheck = a @codemirror/lint source backed by a WASM dictionary on a Web Worker, masked by the Lezer markdown tree so code / links / frontmatter / media: tokens are never checked. The wavy underline is a CM6 mark decoration in --cairn-warning-ink.

State 2 · The spellcheck correction popover
Click the wavy word (or open it from the footer count) and an anchored popover offers suggestions, Add to dictionary (a persistent personal dictionary for names, product terms, slugs), and Ignore (this instance, this session). Number keys pick a suggestion; Escape dismisses and returns focus to the editor. This is the quick single-word fix. Tidy (states 3 to 5) is the deliberate whole-document pass.
The popover is a role="dialog" anchored at the CM6 word range, the same exception MediaInsertPopover earned: Escape + backdrop dismiss, focus restored to the editor. Below the narrow breakpoint it drops the anchor and becomes a bottom sheet.
Add to dictionary writes to a git-committed per-site dictionary file (consistent with content-in-git), so the term never flags again across the site. Ignore is session-local only.

State 3 · Tidy thinking (the Worker calls Claude)
Pressing Tidy opens the panel and POSTs the document (or the selection) to a Cloudflare Worker, which calls Claude with the copy-edit judgment contract. The buffer is untouched. The panel shows an honest loading state with a Cancel; the editor column narrows to make room.
Transport reuses the media passes: a text/plain POST with the CSRF token in X-Cairn-CSRF, parsing a SvelteKit ActionResult envelope. The API key never reaches the browser. Default model Sonnet; Haiku is the cheaper option (settings, state 6).
The Worker returns corrected text (poplar's string-in / string-out model). The CLIENT computes the diff (an LCS over the markdown) and segments it into the cards. The result is validated as a proofread, not a restructure (frontmatter intact, media: / :::figure / code spans byte-identical) before any card is offered.

State 4 · Tidy review (the heart: editor highlights, panel drives)
The panel lists each proposed change as a card: the category (glyph + label, never hue alone), the before/after (deletion struck in error-ink, insertion underlined in positive-ink, the safety contract made visible), a jump-to, and per-card Accept / Reject. The header summarises the run and offers Accept all / Reject all. The active card (grammar) has its locus lit in the editor. The three changes are all corrections, never voice: a doubled word, an its/it's slip, and a consistency harmonised to the author's OWN dominant spelling (they wrote "e-mail" elsewhere).
Panel = role="region" aria-label="Tidy review", non-modal (no scrim, editor stays in the tab order). Focus moves into the panel on open, returns to Tidy on close, window Escape closes. The header summary is a role="status" live region; it announces the count when the run lands.
Keyboard: arrow / Tab between cards, A accept and R reject the focused card (the focused card is the active one and lights its locus), J jump-to, Shift+A Accept all. Every action is reversible per card before the buffer is touched.
Accept-all applies in ONE batched CodeMirror transaction (avoids the freeze-on-many and nag-per-line failures). The original stays in the buffer until accept (the cairn preview-and-confirm idiom). An accepted tidy is a normal git diff on the next save.

State 5 · Tidy resolved (working through, then done)
As the author accepts or rejects, each card collapses to a quiet status line with an Undo, and the editor applies the accepted edit. Here two are accepted and one rejected; the run is complete, so the panel shows its calm done state. Closing returns focus to the Tidy trigger. The accepted edits ride the next save as an ordinary git commit, revertable like any edit.
A resolved card keeps an Undo until the panel closes, so a too-hasty Accept is recoverable in the same session. The done state's summary is the same role="status" region updated, never a new role="alert" (no failure happened).

State 6 · The opt-in and settings surface
The local-versus-remote split is the whole reason one is default and one is opt-in. Spellcheck runs in the browser, leaks nothing, costs nothing, so it ships on. Tidy sends content to the Anthropic API and costs tokens, so the site owner turns it on and supplies a key. The model choice (Sonnet default for the judgment, Haiku cheaper) and the personal dictionary sit with their features.
The key is a Cloudflare Worker secret, never shipped to the browser (the field writes through the same admin-action transport, then shows masked). Tidy's controls stay disabled until a key is set, so the opt-in is honest about its prerequisite.
Settings

Writing help

Two kinds of help for the editor. One runs on your device; one calls an AI you pay for.

Spellcheck

Marks misspelled words as you type. Runs on your device, so nothing leaves the browser.

Check spelling while editing
A wavy underline marks a misspelled word. Code, links, and frontmatter are left alone.
On by default
Personal dictionary
Words you added stay out of the checker. Names, product terms, and slugs add up fast.
cairn Telemark Klister
The dictionary is saved with your site at .cairn/dictionary.txt, so the whole team shares it.

Tidy

A light copy-edit on demand. It fixes errors and keeps your voice. It calls the Anthropic API, so it needs a key and it costs tokens.

Turn on Tidy
Adds the Tidy button to the editor toolbar. Off until you add a key below.
Anthropic API key
The key is stored as a Worker secret, never in your site code or the browser. Get one from the Anthropic console. You pay Anthropic for what Tidy uses.
Model
Sonnet reads your voice more carefully. Haiku is faster and cheaper for plain fixes.
What Tidy will and won't do
It fixes spelling, typos, doubled words, and clear grammar mistakes, and matches a term to how you spelled it. It never rephrases for style, changes your tone, or adds or cuts content.

State 8 · Narrow render (the panel as a bottom sheet)
On a phone the companion cannot sit beside the editor, so it becomes a bottom sheet over the full-width manuscript. The cards are the same. Jump-to scrolls the editor behind the sheet and lights the locus; a drag handle collapses the sheet to a peek so the author can see the lit text, then expands it again.
This is the modal-sizing rule's one sanctioned exception (below the narrow breakpoint a companion may become a bottom sheet), the same shape the media insert popover takes. The sheet keeps the region role and the focus contract.
