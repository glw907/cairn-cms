Editor copy-edit and spellcheck — Direction "review"

Two features for the cairn admin editor. Spellcheck is local, private, and on by default; it stays inline as a wavy underline and a correction popover, because it runs continuously as you type. Tidy is a light, voice-preserving copy-edit powered by Claude; it is opt-in, and it is the one this direction shapes.

In this direction Tidy does not mark up the live manuscript. It steps you into a focused review surface: a git-style diff of the markdown with per-hunk Accept and Reject, Accept-all and Reject-all, and a running tally. The diff shows deletions as well as insertions, so you can see your voice survived. On Confirm the kept changes apply in one transaction and you are back in the editor. This is cairn's deliberate preview-and-confirm pattern, and it is strongest when the edit is large.

The everyday editor. A real post in markdown source: frontmatter, prose, a code span, a media: chip, and a :::figure. Screen 2 · Spellcheck: the correction popover (light)
Spellcheck stays inline. Click the underlined word and a small anchored popover offers the ranked suggestions, then Add to dictionary (for names, product terms, slugs) and Ignore. This is the official CodeMirror lint quick-fix surface in cairn's chrome. No step-in, no diff: a continuous check gets a continuous, lightweight correction.
a11y — the popover is the @codemirror/lint tooltip: keyboard reachable from the diagnostic, arrow keys move the highlighted suggestion, Enter applies, Escape closes and returns the caret to the word. The suggestion list is a real listbox; the two utilities are buttons after it.
engine — the dictionary is a client-side SymSpell WASM worker, markdown-aware via the Lezer tree: the frontmatter keys, the code span, the media: token, and the figure fence are masked, so they are never flagged. "Add to dictionary" appends to a per-site git-committed custom word list (content-in-git), so the team shares one growing dictionary.

Screen 3 · The Tidy review surface — unified diff (light)
The step-in surface. Tidy proposes a light copy-edit and you read it as a diff before anything lands. Each change is a hunk: the removed text struck in red, the new text in green, so you can confirm your voice survived. Accept and Reject sit on each hunk; Accept all and Reject all sit up top with a running tally. The buffer behind this is untouched until you press Apply.
design — following poplar's model, the model returns a plain corrected STRING, not structured edits. cairn computes the diff itself (an LCS over the markdown) and infers each category LOCALLY (typo, grammar, consistency, doubled word) where it can do so safely. The category is a quiet aid, never a claim the model made about its own edit.
engine — before this surface renders, the rewrite is validated as a PROOFREAD, not a restructure: the frontmatter block, the code span, the media: token, and the :::figure directive must be byte-identical in the result. A rewrite that touched any of them is refused and the author is told tidy could not run cleanly, never shown a partial diff.
a11y — the dialog is a native <dialog> opened with showModal() (focus trap + Escape for free). The tally is a single role="status" live region; per-hunk Accept/Reject is the segmented check-and-tint recipe with aria-pressed; j/k move between hunks, a/r toggle the focused one, and the colors are never the only state cue (the gutter glyph and the strike/underline carry it too).

Screen 5 · The split-diff layout (light)
The same changes, read side by side. Before on the left, after on the right. Split helps when a single change is longer than a word; unified is tighter for a run of small fixes. The layout toggle lives in the header and is remembered. Everything else about the surface is the same.
design — both layouts are computed from the ONE local diff; the toggle is pure presentation. Split reuses the same hunk model, the same Accept/Reject state, the same one-transaction Apply. Default is Unified (it suits the common case, several small corrections); the choice persists as cairn-tidy-diff-layout.

Screen 7 · The opt-in and settings surface (light)
Where the two features are governed, in Settings. Spellcheck is on by default because it runs in your browser and sends nothing anywhere. Tidy is off until the site owner turns it on and adds an Anthropic API key, because it sends your draft to the API and costs tokens. The copy says so plainly; the trust split is the reason, not a footnote.
a11y — each feature toggle is a real labelled control with the on/off state in the toggle, not in color. The API-key field is a password input with a reveal; "Saved" is a live status, not a color-only cue. The model choice is a segmented check-and-tint group.
engine — the API key is a Worker secret (it never reaches the browser; tidy runs through the same CSRF-guarded admin action as the media uploads). The personal dictionary is a git-committed per-site file, edited through the same GitHub-App commit pipeline as content.

Writing help

Two ways cairn helps you catch errors while you write. One runs on your computer; one uses Claude.

SpellcheckOn by default
Underlines misspelled words as you type and offers fixes. It runs entirely in your browser, so your writing never leaves the page. It knows markdown, so it skips code, links, frontmatter, and media: tokens.
Site dictionary
Words you add from the popover are saved here and shared with everyone who edits this site. Good for names, places, and product terms.
corduroy klister Telemark ecnordic
TidyOpt-in
A light copy-edit from Claude. It fixes spelling, grammar, and consistency, and leaves your wording and tone alone. You review every change as a diff before it lands. It sends the post you are editing to the Anthropic API and uses tokens, so it is off until you turn it on and add a key.
Anthropic API key
Stored as a Worker secret. It is never sent to the browser. Create one at console.anthropic.com.

Model
A light copy-edit needs judgment, so Sonnet is the default. Haiku is cheaper and faster for short, plain posts.
