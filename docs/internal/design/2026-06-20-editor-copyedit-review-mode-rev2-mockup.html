Tidy review mode — rev.2

This is rev.2 of the focused step-in review surface for tidy, the light, voice-preserving copy-edit from Claude. It starts from the winning spine and folds in the grafts the synthesis asked for, while honoring the brief's lead decisions. The big shift from the spine: tidy no longer guesses a style preference. The changes you see are objective fixes (spelling, doubled words, spacing, clearly-wrong grammar) plus normalizations the site owner switched on (a comma rule, a number style). Regional spelling is never touched.

The safety rank is the spine of rev.2. Objective fixes read quiet and come pre-kept. A change that takes judgment (a normalization, or a grammar fix that reworded a clause) is flagged to review, starts undecided, and is not swept in by Accept fixes until you confirm it. Every normalization carries a plain reason computed from your own text. You step through with the keyboard, you see one line of context around each change, and the whole tidy is one Undo away after it lands.

State 0 · The desk at rest, with the spellcheck underline (light)
The everyday editor, included so the review surface has a surface to step over. The one change from the spine is the squiggle color: per the synthesis (MK-3) the spellcheck underline is now --cairn-warning-ink, and the deletion red is reserved for tidy. Tidy lives in the toolbar as a labelled action, because it is something you invoke, not a format you toggle.
a11y. The misspelling is a @codemirror/lint diagnostic; clicking it opens the correction popover (suggestions, add-to-dictionary, ignore). The underline color is --cairn-warning-ink at severity:'info', the one spellcheck token across the whole feature.

State 1 · The review surface, mixed hunks (light) — the hero
The step-in surface. Tidy proposes a light copy-edit and you read it as a diff before anything lands. The two plain fixes (a misspelling, a doubled word) are quiet and already kept. The two that take judgment (a grammar fix that reworded a clause, and a comma rule the site owner turned on) are flagged to review, start undecided, and each shows why. Accept fixes sweeps only the plain ones; the judgment ones wait for you. The buffer behind is dimmed, not hidden, so you can read the surrounding line and jump to it.
grafts folded here: the safety-ranked categories (objective quiet + pre-kept, judgment flagged + undecided + not in Accept-fixes), the mandatory because-line on every normalization, one context row above and below each hunk, the line-ref as a real "Show in text" affordance, the dimmed (not blurred) buffer, the two-region live model, and the keyboard-hint pill.
a11y. The dialog is a native <dialog> opened with showModal() (focus trap + Escape for free). The tally is one role="status" that updates only on bulk actions; a second aria-live="polite" region (visually hidden) narrates the last single action. Per-hunk Accept/Reject is the segmented check-and-tint recipe with aria-pressed; the gutter glyph and the strike/underline carry state so color is never alone.
engine. Following poplar's model the model returns a plain corrected STRING; cairn computes the diff and infers each category LOCALLY from the diff shape and the enabled config. The because-line for a normalization names ONLY the config setting that authorized it, never a count of the author's own usage (counting the author's own habit is the harmonize-to-author judgment cairn dropped). Before this renders, the rewrite is validated as a PROOFREAD: frontmatter, the code span, the media: token, and the :::figure must be byte-identical, or tidy refuses and the author is told it could not run cleanly. All line refs are computed locally, never trusted from the model.

State 3 · Working (cancelable), and nothing to fix (light)
Two edge states. Left: tidy is running, so the surface opens in place with a calm wait, and Cancel actually stops the call. Right: nothing needed fixing, so tidy says so rather than showing an empty diff. The copy recruits you as the reader of the changes; it does not promise the output is safe to skip.
grafts shown: the cancelable working state wired to a real abort (the AbortController plus a bounded timeout, TD-7), and the genuine "Nothing to fix" empty/clean state that never opens an empty diff (graft 5 / MK-5).
a11y. The working state is a role="status" region, so the wait is announced once. On transition the result ("4 changes to review" or "Nothing to fix") lands in a live region and focus moves to the first hunk or to "Back to editing" (MK-5).

State 4 · Back at the desk, the whole-tidy Undo (light)
After Apply the editor returns to the desk with the changes landed in one move. A quiet "Undo tidy" sits in the save strip, so if a change turns out to have crossed into your voice you take the whole tidy back at once. It is the same register as the save state, and it clears as soon as you type. The accepted edits show in the text; the change you rejected reads as you left it.
graft 6: the surfaced whole-tidy Undo. Apply is one batched view.dispatch({changes}) and one history entry, so a single session-scoped Undo reverts the entire applied tidy. The graft is to NAME it in the desk, not to invent new machinery; ordinary editor Undo already covers it.
