Tidy settings: Direction "list"

This is the settings surface for tidy, the light, voice-preserving copy-edit from Claude. The chosen editor surface is the review-mode mockup; this screen is where an editor governs what tidy is allowed to change before it ever opens that review.

The direction is a grouped toggle list, the standard settings idiom, pushed for legibility and speed. Two clearly separated sections: Fixes, the objective errors that default on, and Style conventions, the choices that default off. Every row carries a tiny live example of the change it makes, so a non-technical editor reads the example, not the rule. A style row with variants reveals an inline segmented control when it is on.

The brief's two-tier split is load-bearing here. The developer turns tidy on and supplies the API key at deploy time; the editor governs the conventions on this screen. When the developer has not enabled tidy, this whole surface is replaced by a short, honest note (screen 3), never a teasing disabled form.

Screen 1 · Editor tier, enabled, at rest (light)
The everyday state. Fixes are all on (they are errors, not choices); Style conventions are all off (cairn leaves the author's choice until someone declares it). The developer tier shows read-only at the top, so the editor sees the feature is live and who controls it. Nothing here demands a decision; the editor can leave this screen exactly as it loads.
a11y. Each row toggle is a real labelled checkbox; the on/off state lives in the control, never in color. The tiny examples are aria-hidden (the row name carries the meaning); a screen reader hears "Spelling and typos, on", not the struck/added runs.
engine. The convention set is stored as committed per-site YAML, edited through the GitHub-App commit pipeline, so it is diffable and shared across editors. The system prompt is built from the enabled conventions only; an off convention contributes no rule line, so the model is never told to touch it.

Screen 2 · A convention on, with its variant open (light)
The active state. Turning a style convention on reveals an inline segmented control for its variants; the example under the control updates to the chosen variant, so the editor sees the exact effect of their pick. This is the only place the list grows in complexity, and only because the editor opted in. One Fix is shown turned off, to show an exception stays legible.
a11y. The variant control is the shipped segmented check-and-tint group: role="group", each segment a button with aria-pressed, the active one marked with a check glyph so hue is never the only cue (WCAG 1.4.1). Roving tabindex carries keyboard traversal. The variant control mounts only when the row's toggle is on, so a screen reader never meets an orphan control.

Screen 3 · The visibility gate · tidy not enabled (light)
What the editor sees when the developer has not turned tidy on. The convention list is gone, not disabled. In its place is one honest note: tidy is a developer setup, here is what it needs, and here is who does it. Spellcheck still works because it is local, so the screen reassures rather than reading as broken. There is no teasing greyed-out form.
a11y. The gate is the empty-state recipe, not a form: no disabled controls live in the tab order, so a keyboard or screen-reader user is never offered config they cannot use. The checklist is read-only text, the one link goes to the docs.
engine. The editor tier renders only when tidy.enabled is true AND the API key secret is present (the brief's MK-4 gate, extended to the whole settings section). When either is missing, the route serves this state instead of the convention list.

Tidy is not set up yet
Tidy uses Claude to copy-edit your drafts, so it sends your writing to the Anthropic API and costs a little per use. That makes it a developer setup, not a switch in here. Once it is on, this page is where you choose what it can change.
1 Turn tidy on in the site config. Set tidy.enabled: true.
2 Add an Anthropic API key as a Worker secret. It stays on the server and never reaches the browser.
Spellcheck is already working. It runs in your browser, so it needs no setup and underlines misspellings as you type.
Screen 4 · Editor tier, at rest, dark
The resting state through the dark tokens. The example washes are color-mixed over the dark positive and error inks, so the before/after contrast holds with no second palette. The Fixes card keeps its faint settled edge; the model control keeps its tint and check.
