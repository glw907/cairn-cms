Editor help · Direction A: “Quiet and woven”
The bet: push help almost entirely into the work surface. There is no standalone office Help view. Getting-started is carried by the teaching empty state, one dismissible “New here? Start here” card, and a resumable progress chip. Daily reference is one context-aware slide-over, summoned by a quiet “?” in the office topbar and by the editor footer’s existing Markdown help link. Two woven touches teach in place: a hint at the point of typing, and a field hint under a frontmatter input. Every surface shows only this site’s concepts (Posts and Pages) and its enabled features, because the admin knows the config at runtime.

Screen 3 · Help woven into the work (light)
The pull-not-push idea, made literal. Left, in the manuscript: the author just typed ## to start a line, so a quiet whisper appears beside that line to confirm what it does. It is not a modal. It fades on the next keystroke and never blocks typing. No major editor teaches markdown at the moment of typing; this is cairn’s opening. Right, in Details: each field carries a short hint under it (the Sanity/Contentful pattern), and a missing social image is a warning the author can publish straight past, never a gate.

How this direction holds the line

Two anchored homes, almost no upfront help.

  • Getting started lives in the teaching empty state, one dismissible card, and a resumable five-step chip. A real sample post defuses the blank page. Nothing tours, nothing chains coachmarks.
  • Daily reference is one context-aware slide-over, summoned by the quiet “?” in the office and the editor’s Markdown help link on the desk. It reuses the Details panel geometry, so it is cairn’s chrome, not a docs site.
  • Woven help teaches at the point of typing and under each field, and shows only this site’s concepts and enabled blocks.
  • Reassurance is true by design: drafts are private until Publish, and Publish is reversible. The copy says so at the save and publish moments, never as filler.
