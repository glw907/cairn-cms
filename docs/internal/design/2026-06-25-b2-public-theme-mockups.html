How a cairn marks the way

A cairn is a pile of stones that marks a path. You add one stone, the next walker adds another, and over a season the route becomes legible to anyone who follows. A site works the same way: each post is a stone, and the theme is the shape they take together.

This page is written in plain Markdown. The heading you just read, the list below, the quote, the code, and the small note boxes are all ordinary text in a file. The editor renders them, the public page renders them the same way, and nothing here was hand-placed in a layout tool.

The manuscript leads

Good reading is mostly restraint. The column holds at about sixty-six characters so your eye finds the next line without hunting, the body sits near nineteen pixels, and the space above a heading is larger than the space below it, so a heading belongs to the text it introduces. You can read this for an hour.

Type is a beautiful group of letters, not a group of beautiful letters.

Matthew Carter

When a single line deserves to ring out, it can step into the margin as a pull-quote without leaving the flow of the text:

Restraint is not the absence of a choice. It is the one choice you trust enough to repeat.

Code, set for reading

Inline bits like render(md) sit in a tinted chip. Fenced blocks are highlighted at build time, so the page ships no highlighter and the colours track the theme:

// the one renderer the editor preview and every page call
export function render(md: string) {
  return pipeline(md, { highlight: true, theme: 'cairn' });
}
