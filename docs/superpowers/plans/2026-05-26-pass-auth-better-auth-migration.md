# Pass AUTH — Migrate cairn-cms auth to better-auth (D1 + magic-link) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace cairn-cms's hand-rolled magic-link/signed-cookie/KV auth with `better-auth` (Drizzle + Cloudflare D1, magic-link plugin), behind a scanner-safe **POST-confirm** flow, with two-tier `owner`/`editor` roles and DB-backed session revocation — shipped to both live sites with no login outage.

**Architecture:** Auth is **engine code** (engine-fat rule), so the better-auth config, the `createAuth(env)` per-request factory, the confirm-flow server logic, the guard helper, and the manage-editors server functions all live in `@glw907/cairn-cms`; each site keeps a thin per-site surface (a D1 binding, the better-auth catch-all route, the confirm/login route shims, committed migration SQL). better-auth has **no built-in scanner-safe step**, so the email links to a cairn **confirm page** (GET, non-consuming) whose "Confirm sign-in" button triggers verification — scanners GET but don't click. The KV editor allowlist becomes the better-auth **`user` table** (`disableSignUp: true` ⇒ user table *is* the allowlist); roles come from the **admin plugin** (`role` column + `revokeUserSessions`). This fixes C1 (single-use on strongly-consistent D1) by construction, C2 (POST-confirm), H3 (lib owns the token/compare lifecycle), and M3 (server-side session revocation).

**Tech Stack:** `better-auth@^1.6` (1.6.11 stable), `drizzle-orm` (D1 dialect) + `better-auth/adapters/drizzle`, `@better-auth/cli` + `drizzle-kit` (schema/migrations), `better-sqlite3` (test + local schema-gen), Cloudflare **D1** + **Workers** (`nodejs_compat`), SvelteKit 2 / Svelte 5, Cloudflare Email Sending (wired into `sendMagicLink`; Resend coded fallback per H6). Verified against the better-auth research report (2026-05; docs at better-auth.com).

**Shippable split:**
- **AUTH-1 (Phases 0–3):** working magic-link login on better-auth/D1 + POST-confirm + guard. Ends with both sites logging in via better-auth (single owner pre-seeded), old auth removed.
- **AUTH-2 (Phases 4–6):** roles via admin plugin, manage-editors rewired to `auth.api`, full allowlist→D1 migration, version bump + both-site repoint + CI green.

**Pre-reqs / what exists (read before starting):**

- [ ] **Step 3: Write `src/lib/auth/config.ts` (the real factory)**

```ts
// cairn-core: the better-auth instance. Auth is engine code (engine-fat rule), so the whole
// config — Drizzle/D1 adapter, magic-link (POST-confirm-shaped send), admin roles, allowlist —
// lives here. Instantiated PER REQUEST in hooks.server.ts (the D1 binding is request-scoped).
import { betterAuth } from 'better-auth';
import { drizzleAdapter } from 'better-auth/adapters/drizzle';
import { drizzle } from 'drizzle-orm/d1';
import { magicLink, admin } from 'better-auth/plugins';
import { APIError } from 'better-auth/api';
import type { D1Database } from '@cloudflare/workers-types';
import { sendMagicLink, type EmailSender } from '../email';

export interface AuthEnv {
  AUTH_DB?: D1Database;
  AUTH_SECRET?: string;
  BETTER_AUTH_URL?: string;
  EMAIL?: EmailSender;
  PUBLIC_ORIGIN?: string;
}

/** Branding the magic-link email needs; threaded from the site adapter via hooks. */
export interface AuthBranding {
  siteName: string;
  sender: string; // from-address
}

/**
 * Build the per-request better-auth instance. `disableSignUp:true` makes the `user` table the
 * editor allowlist (only existing users get a link); a databaseHooks guard belt-and-suspenders
 * blocks creation of non-allowlisted users. The email links to the cairn CONFIRM page (not
 * better-auth's GET verify endpoint), so scanner prefetch can't burn the token (C2 / POST-confirm).
 */
export function createAuth(env: AuthEnv, branding: AuthBranding) {
  if (!env.AUTH_DB) throw new Error('AUTH_DB (D1) binding is required');
  const db = drizzle(env.AUTH_DB);
  const origin = env.PUBLIC_ORIGIN || env.BETTER_AUTH_URL || 'http://localhost';

  return betterAuth({
    appName: branding.siteName,
    secret: env.AUTH_SECRET,
    baseURL: origin,
    trustedOrigins: [origin],
    database: drizzleAdapter(db, { provider: 'sqlite' }),
    plugins: [
      magicLink({
        disableSignUp: true,
        expiresIn: 600,
        storeToken: 'hashed',
        sendMagicLink: async ({ email, token }) => {
          // POST-confirm: link to OUR confirm page carrying only the token; consumption
          // happens when the user clicks "Confirm sign-in" there, never on a scanner GET.
          const link = `${origin}/admin/auth/confirm?token=${encodeURIComponent(token)}`;
          if (!env.EMAIL) throw new Error('EMAIL binding is required to send magic links');
          await sendMagicLink(env.EMAIL, email, link, branding.siteName, branding.sender);
        },
      }),
      admin({ defaultRole: 'editor' }),
    ],
    databaseHooks: {
      user: {
        create: {
          before: async (user: { email: string }) => {
            // Belt-and-suspenders: with disableSignUp the table is the allowlist, but if a
            // create path ever opens, only known emails may land.
            const row = await env.AUTH_DB!
              .prepare('SELECT 1 FROM user WHERE lower(email) = ?')
              .bind(user.email.toLowerCase())
              .first();
            if (!row) throw new APIError('FORBIDDEN', { message: 'This email is not authorized.' });
          },
        },
      },
    },
  });
}

export type Auth = ReturnType<typeof createAuth>;
```

```ts
export async function sendMagicLink(
  sender: EmailSender, to: string, link: string, siteName: string, from: string,
): Promise<void> {
  const expiry = "This link expires in 10 minutes and works only once. If you didn't request it, ignore this email.";
  try {
    await sender.send({
      to, from,
      subject: `Your ${siteName} sign-in link`,
      text: `Sign in to ${siteName}:\n\n${link}\n\n${expiry}`,
      html: `<p>Sign in to ${siteName}:</p><p><a href="${link}">Confirm sign-in</a></p><p style="color:#666;font-size:0.9em">${expiry}</p>`,

    // H6: Email Sending is beta + the sole auth channel. Surface, audit, and (future) fall back to Resend.
    console.error(`magic-link email send failed for ${to}:`, err);
    throw err;
  }
}
```

```ts
import type { Auth } from './config';

export interface CairnUser { id: string; email: string; name: string; role: 'owner' | 'editor'; }

/** Read the better-auth session into a cairn user (or null). */
export async function loadSession(auth: Auth, request: Request): Promise<CairnUser | null> {
  const session = await auth.api.getSession({ headers: request.headers });
  if (!session?.user) return null;
  const u = session.user as { id: string; email: string; name: string; role?: string };
  return { id: u.id, email: u.email, name: u.name, role: u.role === 'owner' ? 'owner' : 'editor' };
}

export function requireSession(user: CairnUser | null): CairnUser {
  if (!user) throw redirect(303, '/admin/login');
  return user;
}

/**
 * POST-confirm verification. Invoked from the confirm page's POST action: it proxies the token
 * to better-auth's verify endpoint via the per-request handler, then forwards the Set-Cookie(s)
 * onto a 303 to /admin. Scanners GET the confirm page (no consume); only this POST consumes.
 */
export async function confirmSignIn(event: { request: Request; locals: { auth: Auth }; url: URL }): Promise<Response> {
  const form = await event.request.formData();
  const token = String(form.get('token') ?? '');
  if (!token) throw redirect(303, '/admin/login?error=expired');
  const verifyUrl = `${event.url.origin}/api/auth/magic-link/verify?token=${encodeURIComponent(token)}&callbackURL=/admin`;
  const res = await event.locals.auth.handler(new Request(verifyUrl, { headers: event.request.headers }));
  const cookies = res.headers.getSetCookie();
  if (!cookies.length) throw redirect(303, '/admin/login?error=expired');
  const headers = new Headers({ location: '/admin' });
  for (const c of cookies) headers.append('set-cookie', c);
  return new Response(null, { status: 303, headers });
}

export async function signOut(event: { request: Request; locals: { auth: Auth } }): Promise<Response> {
  const res = await event.locals.auth.handler(
    new Request(`${new URL(event.request.url).origin}/api/auth/sign-out`, { method: 'POST', headers: event.request.headers }),
  );
  const headers = new Headers({ location: '/admin/login' });
  for (const c of res.headers.getSetCookie()) headers.append('set-cookie', c);
  return new Response(null, { status: 303, headers });
}
```

```ts
import { redirect, error } from '@sveltejs/kit';
import type { Auth } from './config';
import { type CairnUser } from './guard';

export interface AdminsData { admins: CairnUser[]; self: string; saved: boolean; error: string | null; }

export function requireOwner(user: CairnUser | null): CairnUser {
  if (!user) throw error(401, 'Not signed in');
  if (user.role !== 'owner') throw error(403, 'Owner access required');
  return user;
}

type Ev = { locals: { auth: Auth; user: CairnUser | null }; request: Request; url: URL };

export async function adminsLoad(event: Ev): Promise<AdminsData> {
  const owner = requireOwner(event.locals.user);
  const res = await event.locals.auth.api.listUsers({ query: { limit: 200 }, headers: event.request.headers });
  const admins = (res.users ?? []).map((u: any) => ({
    id: u.id, email: u.email, name: u.name, role: u.role === 'owner' ? 'owner' : 'editor',
  })) as CairnUser[];
  return { admins, self: owner.email, saved: event.url.searchParams.get('saved') === '1', error: event.url.searchParams.get('error') };
}

export async function addAdmin(event: Ev): Promise<never> {
  requireOwner(event.locals.user);
  const form = await event.request.formData();
  const email = String(form.get('email') ?? '').trim().toLowerCase();
  const name = String(form.get('name') ?? '').trim();
  const role = form.get('role') === 'owner' ? 'owner' : 'editor';
  if (!/^[^@\s]+@[^@\s]+\.[^@\s]+$/.test(email) || !name) throw redirect(303, `/admin/admins?error=${encodeURIComponent('Enter a valid email and name')}`);
  await event.locals.auth.api.createUser({ body: { email, name, role, password: '' }, headers: event.request.headers });
  throw redirect(303, '/admin/admins?saved=1');
}

export async function removeAdmin(event: Ev): Promise<never> {
  const owner = requireOwner(event.locals.user);
  const form = await event.request.formData();
  const email = String(form.get('email') ?? '').trim().toLowerCase();
  if (email === owner.email) throw redirect(303, `/admin/admins?error=${encodeURIComponent("You can't remove yourself")}`);
  const target = await findUserByEmail(event, email);
  if (!target) throw redirect(303, `/admin/admins?error=${encodeURIComponent('No such editor')}`);
  await event.locals.auth.api.removeUser({ body: { userId: target.id }, headers: event.request.headers });
  throw redirect(303, '/admin/admins?saved=1');
}

export async function setAdminRole(event: Ev): Promise<never> {
  const owner = requireOwner(event.locals.user);
  const form = await event.request.formData();
  const email = String(form.get('email') ?? '').trim().toLowerCase();
  const role = form.get('role') === 'owner' ? 'owner' : 'editor';
  if (email === owner.email && role !== 'owner') throw redirect(303, `/admin/admins?error=${encodeURIComponent("You can't demote yourself")}`);
  const target = await findUserByEmail(event, email);
  if (!target) throw redirect(303, `/admin/admins?error=${encodeURIComponent('No such editor')}`);
  await event.locals.auth.api.setRole({ body: { userId: target.id, role }, headers: event.request.headers });
  // M3: a demoted/removed editor's existing sessions are revoked server-side.
  await event.locals.auth.api.revokeUserSessions({ body: { userId: target.id }, headers: event.request.headers });
  throw redirect(303, '/admin/admins?saved=1');
}

async function findUserByEmail(event: Ev, email: string) {
  const res = await event.locals.auth.api.listUsers({ query: { searchValue: email, searchField: 'email', limit: 1 }, headers: event.request.headers });
  return (res.users ?? [])[0] as { id: string } | undefined;
}
```

## Risks & gotchas to watch during execution
- **`nodejs_compat` is mandatory** for better-auth on Workers — without it the import itself fails. Verify both sites' `wrangler.toml` has it + a recent `compatibility_date`.
- **Per-request `createAuth`** — never a module-level singleton (the D1 binding is request-scoped). The factory is cheap (no I/O on construction).
- **Exact `auth.api` names** (`listUsers`/`createUser`/`removeUser`/`setRole`/`revokeUserSessions`) and the magic-link verify path (`/api/auth/magic-link/verify`) must be confirmed against the installed 1.6.x docs at execution — adjust if the plugin renamed them.
- **`getSetCookie()`** must forward *all* Set-Cookie headers from the proxied verify/sign-out responses; don't collapse to a single `get('set-cookie')`.
- **No login outage:** keep the old auth + AUTH_KV live until Phase 6 prod smoke passes; cut secrets only after. If a site's prod smoke fails, the previous deploy (old auth) is one `git revert` away.
- **Drizzle-orm peer range** — match what better-auth 1.6.x supports; a mismatch surfaces as adapter type errors.
- **better-auth bundle vs Workers size wall** — run `npx wrangler deploy --dry-run` per site after repoint; better-auth + drizzle should stay well under 10 MB (Paid), but confirm (ties to C4/M5).
