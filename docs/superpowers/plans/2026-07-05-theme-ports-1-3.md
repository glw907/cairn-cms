# Theme ports 1-3: AstroPaper, Foxi, hugo-theme-gallery

> The adopted slate's first three, SERIAL by cost (Starlight, the fourth, runs later as its
> own effort with the Topo dividend). Each port runs the visual-fidelity method end to end
> and ends with the chassis harvest (step 7) BEFORE the next port begins — the compounding
> is the design. Launches when the chassis restructure lands; every port builds as a theme
> on the canonical chassis.

## The bars (all binding, from doctrine)

- GLANCE-INDISTINGUISHABLE from the original at normal viewing; the licensed differences
  are the family responsive standard at the extremes and the cairn plumbing underneath.
- MIT license RE-VERIFIED in-repo at port time; attribution headers in the theme files
  plus a credits note — part of each port's definition of done.
- The five-viewport bar (320/390/768/1440/2560), beating the ORIGINAL at 320 and 2560.
- The fresh-context verifier gates every port (reference vs render, separate labeled
  images, device verdicts); the builder's own "matches" is never accepted.
- Each port names its capability test and reports the verdict: AstroPaper = the pure
  theme-seam proof (zero components expected); Foxi = the composed-page/marketing-section
  stress (the composition primitives' first real test); gallery = the media stress (the
  justified grid + lightbox, and the album-vs-Pages content-model question, answered with
  evidence for the contract beat).

## Per port, the method (the visual-fidelity skill, executed)

1. Reference capture: the original theme's live demo, all its distinct page templates,
   both axis widths plus the extremes.
2. Device catalogue (vision agent, the original's stylesheets where public).
3. Build: a new theme on the chassis (examples/<name>-theme in the engine repo, the
   cairn-theme pattern generalized: a theme = chrome + compositions + tokens + component
   looks), with the screenshot-compare loop against the references.
4. Verifier gate: per-template verdicts; STRUCTURAL findings fix before proceeding.
5. The chassis harvest: seams fought, recipes missing, couplings exposed, removal notes
   that lied — LANDED in the chassis before the next port opens; deeper lessons to the
   ledger for the contract beat.
6. The pixel-diff rider: each theme's visual spec at the family widths joins the showcase
   e2e matrix (regen BY FILE PATH).

## Deliverables

Three themes in the engine repo, each glance-verified and credited; the chassis improved
three times over; the capability-test verdicts (especially Foxi's composition answer and
the gallery's album answer) written to the pre-beta ledger for contract beat 2; the
harvest trail the theme-building tutorial will draw on.
