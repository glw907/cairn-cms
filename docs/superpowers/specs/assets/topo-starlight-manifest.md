# Topo — Starlight behavior manifest

The exhaustive enumeration of Starlight's runtime behavior, to be verified against (never
against the build plan) when porting the anatomy into Topo. Each line is one behavior with a
stable ID. Port fidelity is judged line-by-line: a behavior is "ported" only when Topo
reproduces the described state transition, not merely a similar-looking element.

The two load-bearing breakpoints, extracted from the CSS (`min-width` media queries) so the
whole manifest can reference them:

- **`--bp-sidebar` = 50rem (800px):** below it the left sidebar is a drawer and the TOC is a
  mobile disclosure; at/above it the sidebar is a persistent left rail.
- **`--bp-toc` = 72rem (1152px):** below it there is no right rail (TOC lives in the mobile
  disclosure); at/above it the right-rail TOC appears and the layout is three-pane.

## Group A — The sidebar (left navigation tree)

- **S-001** — Tree derivation, autogenerate: with no `sidebar` config, the sidebar is generated
  from the filesystem structure of the docs collection (`treeify` over the locale's routes),
  each entry labelled by the file's frontmatter `title`.
- **S-002** — Tree derivation, manual: a `sidebar` array in config builds the tree explicitly;
  entries are link objects (`{ label, link|slug }` or a bare slug string) or group objects
  (`{ label, items[] }`), and groups may nest groups and links to arbitrary depth.
- **S-003** — Autogenerated group: `{ label, autogenerate: { directory } }` builds a group from
  a directory's files; entries sort alphabetically by file `id` unless overridden.
- **S-004** — Sort override: a page's `sidebar.order` frontmatter (number) reorders it within its
  autogenerated group; `sidebar.label` overrides the displayed label; `sidebar.hidden: true`
  removes it from the sidebar (applies to autogenerated/slug links only, not custom `link`s).
- **S-005** — Group rendering: each group renders as a native `<details>`/`<summary>` disclosure
  with a rotating caret icon (0deg closed, 90deg open, mirrored 180deg in RTL), transition
  `transform 0.2s ease-in-out`.
- **S-006** — Group default-open logic: a group's `<details>` is `open` when EITHER any
  descendant link is the current page (`flattenSidebar(entries).some(isCurrent)`) OR the group's
  `collapsed` option is false. Config `collapsed: true` makes a group start closed; a group
  containing the current page always starts open regardless of `collapsed`.
- **S-007** — Autogenerate collapse: `autogenerate.collapsed: true` collapses generated subgroups
  by default (same open-if-current override as S-006).
- **S-008** — Current-page marking: the link matching the current route gets `aria-current="page"`
  and is styled with accent background + inverted text + weight 600 (the "you are here" pill).
- **S-009** — Group-collapse persistence across navigations: an `<sl-sidebar-state-persist>` custom
  element serializes each group's open/closed boolean into `sessionStorage` key `sl-sidebar-state`
  (an array indexed by restore-point index), keyed by a `hash` of the sidebar's identity.
- **S-010** — Persistence hash guard: restore runs only if the stored `hash` matches the current
  sidebar hash (`getSidebarHash`, a djb2-style hash over concatenated labels+hrefs); a changed
  sidebar structure invalidates stored open-states rather than misapplying them.
- **S-011** — Persistence viewport guard: state restore is skipped entirely unless
  `matchMedia('(min-width: 50em)')` matches — on mobile the sidebar is inside the drawer, so no
  restore.
- **S-012** — Restore points: each group emits an `<sl-sidebar-restore>` (`SidebarRestorePoint`)
  with a `data-index`; on `connectedCallback` it reads the stored boolean for its index and sets
  its enclosing `<details open>` accordingly. Progressive enhancement — all errors swallowed.
- **S-013** — Sidebar scroll-position persistence: the sidebar scroll container (`#starlight__sidebar`)
  scrollTop is stored alongside open-states and restored on the next page load (desktop only),
  briefly parked on `window._starlightScrollRestore` between the two inline scripts.
- **S-014** — Link badge: a sidebar link may carry a `badge` (`{ text, variant, class }`); it
  renders a `<Badge>` after the label. Variants: `note | tip | danger | caution | success |
  default`.
- **S-015** — Group badge: a group's `<summary>` label may also carry a `badge`, rendered after
  the group label text.
- **S-016** — Link custom attributes: sidebar links accept `attrs` (e.g. `target="_blank"`,
  `style`, `class`), spread onto the anchor; autogenerated links merge `autogenerate.attrs` with
  per-page `sidebar.attrs` frontmatter.
- **S-017** — External links: an entry with `{ label, link: 'https://…' }` renders as a plain
  link (no slug inference, no current-page logic).
- **S-018** — Badge/label i18n: badge `text` and link `translations` accept BCP-47-keyed objects;
  autogenerated internal links use the target page's translated `title` by default.
- **S-019** — Sidebar accessibility: the sidebar is a `<nav aria-label>` (localized
  `sidebarNav.accessibleLabel`); `<summary>` markers are hidden (`::marker`/`::-webkit-details-marker`
  `display:none`) in favor of the custom caret.

## Group B — Search (Pagefind modal)

- **S-020** — Provider: default search is Pagefind (`@pagefind/default-ui`), lazily imported on
  `requestIdleCallback` after `DOMContentLoaded`; zero-config, indexes the built static site.
- **S-021** — Open shortcut, trigger button: a header button (`[data-open-modal]`) opens the modal;
  it renders `aria-keyshortcuts="Control+K"` and a `<kbd>Ctrl</kbd><kbd>K</kbd>` hint (hint hidden
  until JS confirms platform).
- **S-022** — Open shortcut, platform detection: an inline script tests `navigator.platform` for
  Apple devices; on Mac/iOS it swaps the hint to `⌘` and sets `aria-keyshortcuts="Meta+K"`.
- **S-023** — Open shortcut, keydown: a global `keydown` listener opens the modal on
  `(metaKey||ctrlKey) && key==='k'`, calls `preventDefault()`, and TOGGLES — pressing the combo
  while open closes it. (Note: Starlight's docs also list `/` as an opener; the shipped source at
  this commit binds only Ctrl/Cmd-K in `Search.astro` — verify `/` against the target release.)
- **S-024** — Modal element: the modal is a native `<dialog>` opened with `showModal()` (top-layer,
  focus-trapped by the platform); the trigger button starts `disabled` and is enabled once the
  custom element upgrades.
- **S-025** — On-open focus + scroll lock: opening sets `data-search-modal-open` on `<body>`
  (which applies `overflow:hidden`) and focuses the Pagefind `<input>`.
- **S-026** — Every-keystroke behavior: Pagefind UI searches incrementally as the user types
  (debounced internally by Pagefind), showing ranked results live; no submit needed.
- **S-027** — Result grouping (sub-results): configured with `showSubResults: true` — each page
  result expands into nested per-heading sub-results, drawn with a tree-diagram connector
  (vertical rule + elbow on the last child), so one page's matching sections group under it.
- **S-028** — Result icons: `showImages: false` (no thumbnails); each top-level result shows a
  document icon, each sub-result a tree-branch glyph.
- **S-029** — Highlighting: matched terms in excerpts render inside `<mark>` styled with weight
  600 and gray text (no highlight background), so the matched substring is bolded.
- **S-030** — Result excerpt: each result shows a context excerpt around the match; long tokens
  wrap (`overflow-wrap:anywhere`).
- **S-031** — Whole-result link: the entire result card is clickable (`.pagefind-ui__result-link`
  spans the card via an `::after` overlay); hover/focus draws an accent outline and accent-low
  background.
- **S-032** — Clear button: the search input shows a clear (×) button; focusing it draws an accent
  outline; clicking clears the query.
- **S-033** — Close via link click: clicking any result link closes the modal (the `onClick`
  handler detects `href` on the target).
- **S-034** — Close via outside click: clicking outside the dialog frame (on the backdrop / body)
  closes the modal.
- **S-035** — Close via Escape: the native `<dialog>` closes on Escape (platform behavior); on
  `close` the body attribute and scroll lock are removed and the outside-click listener detaches.
- **S-036** — Close via Cancel button (mobile): a `[data-close-modal]` "Cancel" button is shown
  only below `--bp-sidebar` (`md:sl-hidden`) and closes the modal.
- **S-037** — No-results state: Pagefind renders its localized "No results for …" message inside
  the results container when a query matches nothing.
- **S-038** — Dev-mode notice: in `import.meta.env.DEV` the modal shows a warning that search is
  unavailable in dev (Pagefind indexes only production builds); Pagefind is not loaded in dev.
- **S-039** — Modal layout responsiveness: below `--bp-sidebar` the dialog is full-screen
  (100% w/h); at/above it it's centered, `4rem` from top, `max-width:40rem`,
  `max-height: calc(100% - 8rem)`, rounded, with a blurred (`backdrop-filter: blur(0.25rem)`)
  translucent backdrop.
- **S-040** — Trailing-slash normalization: when the project's `trailingSlash: 'never'`, result
  URLs are post-processed to strip the trailing slash (`data-strip-trailing-slash`), so links
  match the site's canonical form.
- **S-041** — Exclude a whole page: `pagefind: false` frontmatter drops the page from the index.
- **S-042** — Exclude a page region: `data-pagefind-ignore` on a wrapper omits that subtree from
  indexing while indexing the rest of the page (e.g. banners are auto-ignored).
- **S-043** — Search i18n: Pagefind UI strings are fed from Starlight's `pagefind.*` translation
  keys plus the placeholder from `search.label`.
- **S-044** — Pluggable provider: search can be replaced (Algolia DocSearch via
  `@astrojs/starlight-docsearch`, or community Typesense) through the search-component override
  seam — note only; the port targets the Pagefind default.

## Group C — The right-rail table of contents (scroll-spy)

- **S-045** — Heading levels included: default TOC includes H2 and H3 (`minHeadingLevel: 2`,
  `maxHeadingLevel: 3`); the selector is built dynamically as `h1#_top, :where(h2,h3)[id]`.
- **S-046** — Configurable levels: `tableOfContents: { minHeadingLevel, maxHeadingLevel }` (global
  config or per-page frontmatter) widens/narrows the range; `tableOfContents: false` disables the
  TOC for the page.
- **S-047** — Overview entry: the TOC's first item is a localized "Overview" link to `#_top`
  (the page title, `h1#_top`), so scroll-spy has a top anchor before the first H2.
- **S-048** — Nested rendering: the list nests by heading level (H3s indented under their H2),
  rendered by a recursive `TableOfContentsList`.
- **S-049** — Splash suppression: pages using the `splash` template get no TOC regardless of
  config.
- **S-050** — Scroll-spy mechanism: an `IntersectionObserver` watches content elements; the
  current TOC link is the one whose heading owns the topmost intersecting element, marked with
  `aria-current="true"`.
- **S-051** — Scroll-spy observed set: the observer watches TOC headings, their following siblings
  (excluding any element that itself contains a TOC heading), direct children of
  `.sl-markdown-content` (to catch content before the first subheading), and heading-free direct
  children of `<main>` (e.g. the banner) — so every scroll position maps to some heading.
- **S-052** — Heading resolution: for any intersecting element the code walks the DOM (previous
  siblings, deepest-last-child, then parent) to find the nearest owning heading; reaching the
  content container short-circuits to the page title (`#_top`).
- **S-053** — Scroll-spy top threshold (rootMargin top): intersections begin at
  `navBarHeight + mobileTocHeight + 32px` from the top (nav height plus 2rem padding; mobile-TOC
  summary height is 0 on desktop) — the active-heading line sits just below the sticky header.
- **S-054** — Scroll-spy bottom threshold (rootMargin bottom): the active band is `53px` tall
  (`bottom = top + 53`, "slightly more than the maximum margin-top in Markdown content"); the
  rootMargin bottom is `bottom - clientHeight`, collapsing the observation zone to a thin band near
  the top of the viewport.
- **S-055** — Init timing: initialization is deferred to `requestIdleCallback` (falling back to
  `setTimeout(…,1)`), so scroll-spy wiring never blocks first paint.
- **S-056** — Resize handling: on `window` resize the observer is disconnected immediately, then
  re-created (debounced `200ms`, on idle) with a freshly measured rootMargin — the band tracks a
  changed header height.
- **S-057** — Right-rail presence: the TOC right rail exists only at/above `--bp-toc` (72rem);
  below it the TOC is delivered by the mobile disclosure (Group D). The rail is a fixed,
  independently scrolling `<aside>` with a hairline inline-start border.
- **S-058** — Rail accessibility: the rail is `<nav aria-labelledby>` titled with the localized
  "On this page"; hidden in print (`print:hidden`).

## Group D — Mobile TOC disclosure

- **S-059** — Mobile TOC element: below `--bp-toc` the TOC renders as a `<details>` disclosure
  (`mobile-starlight-toc`) fixed directly under the nav bar, spanning the content column.
- **S-060** — Current-heading readout: the collapsed summary shows the CURRENTLY active heading's
  text in `.display-current`; the mobile subclass overrides `set current` to keep this label in
  sync with scroll-spy as the reader scrolls.
- **S-061** — Caret + open state: the summary has a rotating caret (90deg when open, RTL-mirrored)
  and an accent border when open.
- **S-062** — Close on link click: choosing a TOC entry closes the disclosure (`details.open=false`).
- **S-063** — Close on outside click: clicking anywhere outside the `<details>` closes it.
- **S-064** — Close on Escape + focus return: Escape closes the disclosure; if focus was inside, it
  returns focus to the `<summary>`.
- **S-065** — Dropdown scroll: the open dropdown caps at
  `calc(85vh - navHeight - mobileTocHeight)` and scrolls internally with `overscroll-behavior:
  contain`.

## Group E — Three-pane responsive collapse

- **S-066** — Desktop three-pane (≥`--bp-toc`, 72rem): left sidebar rail + centered content column
  (clamped to `--sl-content-width`) + right TOC rail; the middle column is width-computed so
  content stays centered in the space between the two rails.
- **S-067** — Two-pane band (`--bp-sidebar`…`--bp-toc`, 50–72rem): left sidebar rail + content;
  NO right rail — the TOC is the mobile disclosure at the top of content.
- **S-068** — Single-column mobile (<`--bp-sidebar`, <50rem): no persistent sidebar; content is
  full-width under the fixed header; the sidebar is a drawer (Group F) and the TOC is the mobile
  disclosure.
- **S-069** — Sidebar as drawer (mobile): below 50rem the sidebar pane is `position:fixed`, full
  width, `visibility:hidden` by default, revealed (`--sl-sidebar-visibility: visible`) only when
  the menu toggle's sibling `[aria-expanded='true']` is set.
- **S-070** — Sidebar as rail (desktop): at/above 50rem the sidebar pane is always visible, width
  `--sl-sidebar-width`, with sidebar background and an inline-end hairline; the toggle button is
  hidden (`md:sl-hidden`).
- **S-071** — Fixed header: the header is `position:fixed`, full width, height `--sl-nav-height`,
  above the panes (`--sl-z-index-navbar`); content is padded down by nav height (+ mobile-TOC
  height on mobile).
- **S-072** — Body layout hooks: `<html>`/`<body>` carry `data-has-sidebar` and `data-has-toc`
  attributes that gate the layout math (header end-padding, content centering) — a page with no
  TOC or no sidebar recomputes the column widths.
- **S-073** — Menu-toggle button: below 50rem a fixed circular button (top-right of the header)
  toggles the drawer; `aria-expanded`, `aria-controls="starlight__sidebar"`, and swaps a
  bars/close icon by expanded state.
- **S-074** — Drawer body scroll lock: opening the drawer sets `data-mobile-menu-expanded` on
  `<body>` (`overflow:hidden`); the lock is released at/above 50rem even while set.
- **S-075** — Drawer focus containment: opening the drawer marks `.main-frame` and `.sl-skip-link`
  `inert`, trapping keyboard focus within the header + menu.
- **S-076** — Drawer Escape: Escape (keyup, `code==='Escape'`) closes the drawer and returns focus
  to the toggle button.
- **S-077** — Viewport-change reset: crossing 50rem (`matchMedia('(min-width:50em)') change`) forces
  the drawer to collapsed state (`setExpanded(false)`), clearing inert/scroll-lock so a rotate or
  resize can't strand the UI mid-drawer.

## Group F — Prev/next pagination

- **S-078** — Derivation: prev/next are the sidebar-adjacent links — the sidebar is flattened to a
  link list, the current page located by `isCurrent`, and `entries[i-1]`/`entries[i+1]` become
  prev/next (group boundaries are transparent; it walks the whole flattened order).
- **S-079** — Edge pages: the first page has no prev (index-1 is undefined → omitted); the last
  page has no next; a page not present in the sidebar (`currentIndex === -1`) gets neither.
- **S-080** — Frontmatter override, disable: `prev: false` / `next: false` removes that link even
  when one would derive.
- **S-081** — Frontmatter override, relabel: `prev: 'Custom label'` (string) keeps the derived
  target but changes the visible label.
- **S-082** — Frontmatter override, replace: `prev: { link, label }` sets an explicit target and
  label, overriding derivation.
- **S-083** — Global disable: `pagination: false` in config drops prev/next site-wide.
- **S-084** — Rendering: prev/next render as two cards in an auto-fit grid; prev is left-aligned
  with a leading arrow, next is right-aligned with a trailing arrow (arrows swap in RTL,
  `left-arrow`↔`right-arrow`); each shows a small "Previous/Next" label over the page title.
- **S-085** — Print: pagination is hidden in print (`print:hidden`).

## Group G — Theme switching (light / dark / auto)

- **S-086** — Options: the theme select offers Dark, Light, and Auto (`auto` selected by default);
  it's a styled native `<select>` with a laptop icon and localized labels.
- **S-087** — Persistence key: preference is stored in `localStorage` under `starlight-theme`;
  `light`/`dark` are stored verbatim, `auto` is stored as an empty string (absence ⇒ auto).
- **S-088** — Applied attribute: the resolved theme is written to
  `document.documentElement.dataset.theme` as `light` or `dark` (never `auto` — auto resolves to a
  concrete value).
- **S-089** — Auto resolution: `auto` resolves via `matchMedia('(prefers-color-scheme: light)')`
  to the system scheme.
- **S-090** — Live system-change tracking: a `prefers-color-scheme` `change` listener re-applies
  the theme only when the stored preference is `auto`, so a system dark/light flip updates an
  auto-mode page live.
- **S-091** — FOUC avoidance: an inline `ThemeProvider` script (in `<head>`, before paint) reads
  the stored/system theme and sets `data-theme` immediately; the select is synced afterward via
  `StarlightThemeProvider.updatePickers()`.
- **S-092** — Picker icon sync: changing the select swaps its leading icon (sun/moon/laptop) to
  match the chosen mode, cloned from a hidden `<template id="theme-icons">`.
- **S-093** — Multi-picker sync: `updatePickers` updates every `<starlight-theme-select>` on the
  page (mobile footer + header can both host one), keeping them consistent.

## Group H — Keyboard, focus, and accessibility affordances

- **S-094** — Skip link: a visually-hidden "Skip to content" link is the first focusable element;
  on focus it appears (accent background, top-left) and targets the page title id (`#_top`).
- **S-095** — Skip-link inerting: while the mobile drawer is open the skip link is marked `inert`
  (see S-075) so it can't be tabbed to behind the drawer.
- **S-096** — Search modal focus trap: the native `<dialog>.showModal()` traps Tab focus inside the
  modal and restores focus to the opener on close (platform behavior).
- **S-097** — Drawer focus trap: see S-075 — `inert` on the main frame contains focus in the
  header/menu while the drawer is open.
- **S-098** — TOC/menu Escape semantics: Escape closes the search modal (S-035), the mobile TOC
  (S-064), and the mobile drawer (S-076), each returning focus sensibly.
- **S-099** — Reduced motion / transitions: caret rotations use short CSS transitions; there is no
  JS-driven animation that ignores `prefers-reduced-motion` in the core nav (verify Topo keeps
  transitions CSS-only so the media query can suppress them).
- **S-100** — Landmark labelling: sidebar nav, TOC nav, and dialogs all carry localized
  `aria-label`/`aria-labelledby`; `aria-current` marks both the active sidebar link (`page`) and
  the active TOC link (`true`).

## Group I — i18n / RTL hooks (note only)

- **S-101** — Direction attribute: the route carries a `dir` (`ltr`/`rtl`); components read it and
  the layout uses logical properties (`inset-inline-*`, `padding-inline-*`) throughout, so RTL is
  a data flip, not a restyle.
- **S-102** — RTL element mirroring: carets, pagination arrows, the mobile-TOC caret, and the
  Pagefind page/tree icons are explicitly mirrored under `[dir='rtl']`.
- **S-103** — Localized strings: every user-facing string (labels, "On this page", "Overview",
  "Previous/Next", "Skip to content", search placeholder, theme options) comes from the
  translation system (`Astro.locals.t`); a locale switch re-labels the whole chrome, and
  autogenerated sidebar/pagination labels use per-locale page titles.
- **S-104** — Multi-locale routing: `getLocaleRoutes` builds a per-locale sidebar; the port need
  only preserve the seam (Topo's first consumers are single-locale) — note, not a build target.

## Group J — The 404 page

- **S-105** — 404 route: Starlight ships a prerendered `404.astro` that renders the standard
  `CommonPage` chrome (header, sidebar, theme, search) around a localized "404 — Not found"
  body, so a missing URL stays inside the themed shell rather than a bare server 404.
- **S-106** — 404 content override: the 404 body/title is overridable via the site's translations
  and content; the port must supply a Topo-voiced 404 that reuses the full chrome (the family
  themed-404 pattern already in the chassis).

## Group K — Content components (what a docs page composes)

- **S-107** — Asides (callouts): `<Aside type>` with variants `note | tip | caution | danger`
  (default `note`); each has a variant color, a default localized title, a leading variant icon,
  an optional custom `title`, and an optional custom `icon`. Also available via the
  `:::note[title]` Markdown directive.
- **S-108** — Tabs: `<Tabs>`/`<TabItem label icon>` render a tablist; keyboard arrow-key roving
  between tabs, one panel visible at a time.
- **S-109** — Tab sync: `<Tabs syncKey>` synchronizes selection across every Tabs group sharing the
  key on the page (and persists the choice) — selecting "npm" in one switches all.
- **S-110** — Code blocks (Expressive Code): fenced code renders through Expressive Code with a
  frame, optional title/filename, language, and a COPY BUTTON in the top-right (appears on hover),
  copying the block's text.
- **S-111** — Code block features: line highlighting (`{1,3-5}`), diff markers (`ins`/`del`), text
  marking, word wrap toggle, and terminal-vs-code frame styles — the "polish floor" the brief
  cites from Mintlify.
- **S-112** — `<Code>` component: renders a code block from a `code` prop (for dynamic/imported
  code) with the same Expressive Code treatment.
- **S-113** — Cards / CardGrid: `<Card title icon>` boxes; `<CardGrid>` lays them in a responsive
  grid; `<CardGrid stagger>` vertically offsets alternating columns.
- **S-114** — LinkCard: `<LinkCard title description href>` a whole-card link with a trailing arrow
  affordance.
- **S-115** — LinkButton: `<LinkButton href variant icon>` a call-to-action button; variants
  `primary | secondary | minimal`; optional leading/trailing icon.
- **S-116** — Badge: `<Badge text variant size>` inline pill; variants
  `note | tip | danger | caution | success | default`; sizes `small | medium | large`.
- **S-117** — Icon: `<Icon name size color>` from Starlight's built-in icon set (the same set the
  chrome uses for carets, arrows, magnifier, bars/close, sun/moon/laptop).
- **S-118** — FileTree: `<FileTree>` renders a nested directory listing from a Markdown list;
  `**bold**` highlights an entry, trailing text becomes a comment, directory icons are inferred.
- **S-119** — Steps: `<Steps>` wraps an ordered list into a numbered vertical step sequence with
  connector styling.
- **S-120** — Anchor headings: content headings get hover-revealed anchor links
  (`AnchorHeading` / `.sl-heading-wrapper`) — the "anchor affordance" the brief calls for; each
  heading has a stable `id` used by both the anchor and scroll-spy.
- **S-121** — Markdown default styling + escape hatch: Starlight applies default spacing/typography
  to Markdown output (`.sl-markdown-content`); a `not-content` class opts a subtree out of those
  styles.

## Group L — Page chrome and layout frontmatter

- **S-122** — Banner: a site- or page-level `banner.content` (HTML) renders a centered accent bar
  above the content; it is `data-pagefind-ignore` (never indexed) and balances its text.
- **S-123** — Page title: the H1 is rendered from frontmatter `title` with a stable id (`_top`,
  the skip-link and Overview target); required field.
- **S-124** — Hero: `hero` frontmatter (title/tagline/image/actions) renders a landing hero,
  typically with the `splash` template.
- **S-125** — Splash template: `template: 'splash'` is a wide, TOC-less, sidebar-optional landing
  layout (vs the default doc layout).
- **S-126** — Last-updated: `lastUpdated` (bool/date, global or per-page) renders a localized
  "Last updated" date in the footer, derivable from Git history.
- **S-127** — Edit link: `editUrl`/`editLink.baseUrl` renders an "Edit page" footer link to the
  source; per-page `editUrl` can override or disable.
- **S-128** — Draft pages: `draft: true` excludes a page from production builds (shown in dev with
  a notice).
- **S-129** — Fallback-content notice: for an untranslated page served from the default locale, a
  localized "this page is not yet translated" notice renders (i18n seam) — note only for
  single-locale Topo.
- **S-130** — TOC-per-page override: `tableOfContents` frontmatter overrides the global TOC config
  per page (levels or `false`); `sidebar`, `hero`, `template`, `prev`/`next`, `banner`,
  `pagefind` are the other layout-affecting frontmatter fields enumerated above.

## Hardest-to-port behaviors (build-risk callouts)

1. **Scroll-spy thresholds (S-050–S-056).** The active-heading logic is a thin 53px
   IntersectionObserver band offset by a live-measured header height, plus a DOM-walking
   "nearest owning heading" resolver over a hand-tuned observed set. Reproducing the *feel*
   (which heading lights up at which scroll position) requires porting the rootMargin math and
   the observed-element selectors, not just "observe the headings."
2. **Sidebar group-collapse + scroll persistence (S-009–S-013).** Session-storage state keyed by
   a structural hash, gated to desktop, restored through per-group `<sl-sidebar-restore>` custom
   elements at connect time, all as swallow-all-errors progressive enhancement. Easy to
   half-port (visual collapse works, persistence silently doesn't) and hard to verify.
3. **Two independent responsive breakpoints (S-066–S-077).** 50rem (sidebar rail↔drawer) and
   72rem (TOC rail present↔mobile disclosure) are *separate* thresholds producing a three-pane /
   two-pane / one-pane sequence, with body-attribute-driven column-width math and a
   viewport-crossing reset that clears drawer state. The family five-viewport standard must be
   composed at both breakpoints, not just the phone and desktop extremes.
4. **Search modal fidelity (S-020–S-040).** The behavior is Pagefind's default UI wrapped in a
   native `<dialog>` with platform focus-trap, sub-result grouping with tree connectors, live
   keystroke search, and dev-mode absence. Topo will re-skin Pagefind's DOM; matching the result
   grouping, highlighting, and every close path (link/outside/Escape/Cancel/toggle-key) is the
   detailed work, and Pagefind only exists in production builds so it can't be verified in dev.
5. **Focus/inert choreography across three overlays (S-075–S-098).** The drawer inerts the main
   frame + skip link, the dialog traps focus in the top layer, and Escape has three distinct
   context-sensitive targets. Getting focus return correct on every close, without a focus trap
   leaking or stranding, is the accessibility work that a glance-test never catches.
