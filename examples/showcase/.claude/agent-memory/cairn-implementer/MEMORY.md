# cairn-implementer memory index

- [SiteHeader nav flex-wrap fix](site-header-nav-flex-wrap.md): the 320px wordmark-wrap/nav-clip defect; SiteFooter had the outer flex-wrap precedent; `site-nav` class collides between header/footer in Playwright locators; a touch-target bump changes the header's tallest element and drifts scroll-padding-top + site-visual baselines.
- [Fluid root scaling (ultrawide)](fluid-root-scaling-ultrawide.md): the clamp() formula (round down, not nearest, so the floor breakpoint is pixel-exact); a cross-session screenshot diff is not proof of a regression, only a same-session before/after git-stash pair is.
- [git commit pathspec + concurrent sites](git-commit-pathspec-and-concurrent-sites.md): `git commit -- <path>` commits unstaged hunks too, not just what you staged; the two consumer-site repos can carry a live concurrent session on a non-main branch mid-task, so check `git branch`/`git status` before editing, not just before committing.
- [Consumer-site verification against unpublished engine](consumer-site-verification-against-unpublished-engine.md): `npm pack` + tarball-install a site's cairn-cms dep to prove an unpublished change, then revert package.json/lock; avoid `npm link` (dual-package-hazard false build failures, e.g. vfile-message version split).
- [Reverting a landed fix to prove a test, and stale visual baselines](temporarily-reverting-a-landed-fix-to-prove-a-test.md): `git show <commit>~1:<path>` + `cp` round-trip to watch-it-fail an already-landed fix; a layout-height change makes `site-visual.spec.ts` screenshots stale until the CI regen dispatch, not a red gate you caused.
- [Consumer-site build gate + poisoned node_modules](consumer-site-build-gate-and-poisoned-node-modules.md): a `+server.ts` export outside SvelteKit's allow-list passes check+test but fails `npm run build`; a prior task's packed-tarball verification can leave `node_modules` silently mismatched with the committed lockfile, `rm -rf node_modules && npm ci` before trusting a site's gate.
